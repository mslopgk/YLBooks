# 부록 C · 자주 나는 오류와 해결

!!! note "이 부록에서 확인하는 것"
    - 실습 중 자주 만나는 **화면 증상**을 모았습니다
    - 증상 → 원인 → 해결을 **표로** 바로 찾아볼 수 있게 정리했습니다
    - 표에 없는 문제는 언제나 **5분 룰**(2차시)을 쓰세요

!!! tip "💡 5분 룰을 잊지 마세요"
    5분 동안 안 풀리면, 화면에 뜬 글자를 그대로 복사해서 AI에게 붙여넣고 **"이거 왜 이래요?"**라고 물어보세요. 아래 표에 없는 오류도 대부분 이렇게 풀립니다.

---

## 증상 → 처방 표

| 증상 | 원인 | 해결 방법 | 자주 나오는 곳 |
|---|---|---|---|
| 터미널 맨 앞에 `PS`가 없다 / `irm이(가) ... 인식되지 않습니다` | **파워셸이 아니라 옛날식 창(명령 프롬프트, CMD)**을 열었음 | 창을 닫고 `Windows`+`X` → **[터미널]** 또는 **[Windows PowerShell]**을 다시 열기. 맨 앞 `PS` 확인 | 3차시, 6차시 |
| `node -v`가 "인식할 수 없는 명령"이라고 나온다 | Node.js를 **설치한 뒤에도 옛날 터미널을 그대로** 쓰고 있음 | 열려 있던 터미널을 **모두 닫고 새 터미널**을 열어 다시 시도 | 5차시 |
| `claude` 로그인은 됐는데 계속 못 쓰거나 사용량 관련 안내만 뜬다 | **무료 계정으로는 Claude Code**를 제대로 쓸 수 없음 | `claude.com`에서 **Pro 이상 유료 구독**으로 업그레이드 후 다시 시도 | 6차시 |
| AI가 자꾸 "이렇게 해도 될까요?"라고 묻고 멈춰 있다 | 오류가 아니라 **정상 동작**(승인 대기) | 물어보는 내용을 읽고 괜찮으면 **엔터(승인)**만 누르면 진행됨 | 6, 7차시 |
| `npx expo start --web` 실행 중 "네트워크 액세스를 허용할까요?" 창이 뜬다 | 윈도우 **방화벽**이 Expo의 인터넷(로컬 네트워크) 사용을 막으려 함 | **[액세스 허용]**을 눌러야 브라우저에서 게임이 열림 | 11차시 |
| `npx expo start --web` 실행 후 브라우저가 안 열리거나 흰 화면만 나온다 | 서버가 아직 준비 중이거나, 브라우저가 자동으로 안 열림 | 잠깐 기다린 뒤, 터미널에 보이는 주소를 **직접 브라우저 창에 입력** | 11차시 |
| 게임 화면에서 방향키를 눌러도 반응이 없다 | 브라우저 창의 **포커스**(지금 입력을 받는 곳)가 게임이 아닌 다른 곳에 가 있음 | 게임이 뜬 브라우저 화면을 **마우스로 한 번 클릭**한 뒤 다시 방향키 시도 | 13, 18차시 |
| 방향키를 눌렀는데 화면만 스크롤되거나 글자가 입력된다 | **한글 입력기(IME)**가 켜져 있거나, 다른 입력 칸에 포커스가 있음 | 키보드 오른쪽 아래 **한/영** 키로 영문 상태 확인 → 게임 화면 다시 클릭 | 13, 18, 19차시 |
| 키링 키보드가 블루투스 목록에 안 보인다 | 키링의 **페어링 모드**가 꺼져 있거나, 이미 다른 기기와 연결돼 있음 | 키링의 페어링 버튼/스위치 다시 확인, 휴대폰 등 **다른 기기의 블루투스는 꺼두기** | 17차시 |
| 키링을 연결했는데 눌러도 게임이 반응 없다 | 컴퓨터는 키링을 키보드로 인식했지만, **게임 화면 포커스**가 안 잡혀 있음 | 먼저 메모장에서 눌러 **방향키 반응**을 확인 → 게임 화면 클릭 후 재시도 | 17, 18차시 |
| 화면 글자가 물음표·네모(□)로 깨져 보인다 | 터미널 글꼴·인코딩 문제(일시적) | 창을 닫고 **새 터미널**로 재시도, 안 되면 화면을 그대로 캡처해 AI에게 물어보기 | 전 차시 |
| 폴더나 프로젝트 이름을 한글로 지었더니 이상하게 동작한다 | 한글·띄어쓰기가 들어간 이름을 일부 프로그램이 못 알아들음 | 폴더·프로젝트 이름은 **영문+붙임표**(예: `rhythm-game`)로 다시 만들기 | 4차시, 11차시 |
