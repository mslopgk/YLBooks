# 7차시 · Claude Code 길들이기

!!! note "이번 차시에 하는 일"
    - 매번 처음부터 설명하지 않도록 **프로젝트 메모(CLAUDE.md)**를 만듭니다
    - 자주 쓰는 부탁을 **나만의 짧은 명령**으로 저장해 둡니다
    - **스킬 · MCP · 서브에이전트** — 조금 더 편해지는 기능 세 가지를 한 입씩 맛봅니다

> ⏱️ 걸리는 시간: 약 25분 · 🧰 준비물: 6차시에서 설치한 Claude Code, 그때 만든 작업 폴더

---

## 왜 이걸 하나요?

Claude Code를 새로 켤 때마다 "나 리듬게임 만들 건데, Expo로 만들고, 방향키로 조작할 거야…"를 또 설명해야 한다면 번거롭겠지요. 사람도 새로 온 직원에게 **인수인계 메모**를 남겨두면 매번 설명할 필요가 없어지는 것처럼, Claude Code에게도 그런 메모를 남길 수 있습니다. 그리고 자주 하는 부탁은 짧은 이름 하나로 저장해두면 다음부터는 길게 말할 필요가 없어집니다. 이번 차시에서는 이런 "길들이기" 기술을 배웁니다.

!!! tip "💡 이번 차시는 실습이 짧습니다"
    프로그램을 새로 설치하지 않습니다. 6차시에서 이미 설치한 Claude Code를 **더 편하게 쓰는 법**만 배웁니다. 지금 부담스러우면 가볍게 읽고 지나가도 괜찮습니다. 나중에 필요할 때 다시 펴 보시면 됩니다.

---

## 따라 하기

### 단계 ① 프로젝트 메모(CLAUDE.md)를 만들어 봅니다

작업 폴더에서 `claude`를 실행한 상태로, 아래 명령을 입력해 보세요. 이건 프롬프트가 아니라 Claude Code에 미리 정해진 **짧은 명령(슬래시 명령)**입니다. `/init`을 입력하면 AI가 내 폴더를 스스로 훑어보고, 어떤 프로젝트인지 요약해서 **`CLAUDE.md`**라는 메모 파일을 하나 만들어 줍니다.

!!! quote "🗣️ 이대로 입력해 보세요"
    ```
    /init
    ```

<!-- FIG: id=c07-f01 | type=스크린샷 | src=manual | status=todo | file=images/c07/c07-f01.png -->
> **그림 7.1 — `/init` 실행 화면**
>
> *[캡처 예정(저자): 터미널에서 `/init` 입력 후 CLAUDE.md 생성 완료 메시지. 개인정보 없는 상태로.]*

이제 이 폴더에는 **`CLAUDE.md`**라는 메모 파일이 생겼습니다. 이 파일은 Claude Code가 **매번 대화를 시작할 때 자동으로 읽는** 메모입니다. 즉, 다음에 `claude`를 다시 켜도 "이 프로젝트가 뭐였는지" 처음부터 설명할 필요가 없습니다.

### 단계 ② 메모에 내가 원하는 습관을 추가로 부탁합니다

`/init`이 만들어 준 메모는 기본적인 내용뿐입니다. 여기에 "앞으로 이렇게 대해줬으면 좋겠다" 하는 나만의 규칙을 프롬프트로 덧붙일 수 있습니다.

!!! quote "🗣️ 이대로 복사해서 붙여넣으세요 (AI에게 하는 말)"
    ```
    앞으로 나랑 대화할 때는 항상 쉬운 한국어로,
    전문용어 없이 설명해줘.
    그리고 이 프로젝트는 방향키(← ↓ ↑ →)로 조작하는
    리듬게임이라는 것도 CLAUDE.md에 같이 적어서 기억해둬.
    ```

<!-- FIG: id=c07-f02 | type=스크린샷 | src=manual | status=todo | file=images/c07/c07-f02.png -->
> **그림 7.2 — 메모 파일(CLAUDE.md)을 파일 탐색기로 열어본 모습**
>
> *[캡처 예정(저자): 4차시에서 배운 파일 탐색기로 작업 폴더를 열어, 새로 생긴 CLAUDE.md를 메모장으로 열어본 화면.]*

!!! warning "⚠️ 조심 — CLAUDE.md는 코드가 아니라 그냥 '글'입니다"
    이 파일을 열어봐도 무섭게 생긴 코드가 아니라 **평범한 한글 문장**이 적혀 있을 뿐입니다. 겁먹지 말고 한번 열어서 읽어 보세요. 마음에 안 드는 문장이 있으면 "이 부분 이렇게 고쳐줘"라고 프롬프트로 부탁하면 됩니다.

### 단계 ③ 자주 쓰는 부탁을 나만의 명령으로 저장합니다

작업을 하다 보면 "지금까지 바뀐 걸 실행해서 화면에 잘 나오는지 확인해줘" 같은 말을 계속 반복하게 됩니다. 이런 말을 매번 길게 치는 대신, **짧은 이름 하나**로 저장해 둘 수 있습니다. 저장도 프롬프트로 부탁하면 AI가 알아서 만들어 줍니다.

!!! quote "🗣️ 이대로 복사해서 붙여넣으세요 (AI에게 하는 말)"
    ```
    내가 앞으로 자주 쓸 부탁이 있어.
    "지금까지 바뀐 부분을 실행해서 화면에 잘 나오는지 확인해줘"
    라는 말을, /확인 이라는 짧은 명령으로 저장해줘.
    다음부터는 내가 /확인 이라고만 입력하면 되게 해줘.
    ```

<!-- FIG: id=c07-f03 | type=스크린샷 | src=manual | status=todo | file=images/c07/c07-f03.png -->
> **그림 7.3 — 나만의 명령이 만들어진 뒤, `/확인`을 입력해 본 화면**
>
> *[캡처 예정(저자): 위 부탁 후 실제로 /확인 입력 → AI가 확인 작업을 실행하는 장면.]*

이제부터는 긴 문장을 다시 안 쳐도, `/확인`이라고만 입력하면 똑같은 부탁이 전달됩니다. 이렇게 저장된 짧은 명령을 **슬래시 명령(slash command)**이라고 부릅니다. 앞으로 반복되는 부탁이 생기면 그때그때 "이거 명령으로 저장해줘"라고 하면 됩니다.

!!! warning "⚠️ 조심 — 정식 프로그램 명령어는 아닙니다"
    여기서 만든 `/확인` 명령은 Claude Code 프로그램 자체에 원래부터 있던 진짜 명령어(예: `/exit`)로 등록되는 것은 아닙니다. AI에게 **"내가 앞으로 `/확인`이라고 부르면 이 부탁으로 알아들어 줘"** 하고 `CLAUDE.md` 파일에 메모로 적어 약속해 두는 원리입니다. 그래서 AI와 대화 중일 때만 동작합니다.

!!! tip "💡 어떤 명령이 있는지 잊어버렸을 땐"
    `/help`라고 입력하면 지금 쓸 수 있는 명령들이 목록으로 나옵니다. `/init`, `/exit`처럼 원래 있던 명령과 내가 새로 만든 명령이 함께 보입니다.

## 조금 더 편해지는 기능 세 가지, 한 입씩 맛보기

여기서부터는 지금 당장 다 쓸 필요는 없습니다. "이런 게 있다"는 정도만 알아두면, 나중에 필요할 때 꺼내 쓸 수 있습니다.

### 단계 ④ 스킬 — AI가 꺼내 쓰는 "전문 설명서 꾸러미"

**스킬(Skill)**은 특정 작업을 잘 해내는 방법이 적힌 "전문 설명서 꾸러미"입니다. 예를 들어 "문서를 예쁘게 만드는 법", "발표 자료를 만드는 법" 같은 노하우가 미리 정리되어 있어서, 필요한 상황이 오면 AI가 스스로 꺼내서 참고합니다. 우리는 그냥 평소처럼 부탁만 하면 됩니다.

!!! quote "🗣️ 이대로 입력해 보세요"
    ```
    지금 사용할 수 있는 스킬이 어떤 게 있는지 알려줘.
    ```

### 단계 ⑤ MCP — 바깥세상 도구와 연결하는 통로

**MCP**는 Claude Code가 내 컴퓨터 안뿐 아니라, **바깥의 다른 서비스**(예: 구글 드라이브, 메모 앱, 인터넷 검색 등)와도 연결될 수 있게 해 주는 통로입니다. 쉽게 말해 "AI의 손을 컴퓨터 밖까지 뻗어주는 전선"이라고 생각하면 됩니다. 지금 뭐가 연결되어 있는지는 이렇게 확인할 수 있습니다.

!!! quote "🗣️ 이대로 입력해 보세요"
    ```
    /mcp
    ```

<!-- FIG: id=c07-f05 | type=스크린샷 | src=manual | status=todo | file=images/c07/c07-f05.png -->
> **그림 7.5 — `/mcp` 입력 후 연결 현황 화면**
>
> *[캡처 예정(저자): /mcp 실행 결과. 처음엔 "연결된 것 없음"으로 나올 수 있음.]*

우리 책에서는 MCP를 직접 연결하지 않아도 됩니다. 게임 만들기에는 필요 없기 때문입니다. 다만 "AI 도구가 다른 서비스와도 연결될 수 있다"는 것만 알아두세요.

### 단계 ⑥ 서브에이전트 — 전문 분야만 맡는 보조 AI

**서브에이전트(subagent)**는 주된 AI가 특정 작업만 따로 맡기는 **보조 AI**입니다. 예를 들어 "코드 검토만 담당하는 보조", "오류만 찾아내는 보조"처럼, 한 가지 일에 집중하는 조수를 여러 명 둘 수 있습니다. 마치 팀장(주 AI)이 필요할 때 전문 담당자를 부르는 것과 비슷합니다.

!!! quote "🗣️ 이대로 입력해 보세요"
    ```
    /agents
    ```

<!-- FIG: id=c07-f06 | type=스크린샷 | src=manual | status=todo | file=images/c07/c07-f06.png -->
> **그림 7.6 — `/agents` 입력 후 화면**
>
> *[캡처 예정(저자): /agents 실행 결과 화면.]*

이것도 지금 당장 만들지 않아도 됩니다. "필요하면 전문 보조를 따로 부를 수 있다"는 정도만 기억해 두세요.

!!! info "🔎 낱말 사전"
    - **서브에이전트** — 특정 작업만 전담하는 보조 AI. 주 AI가 필요할 때 불러서 씁니다.

---

!!! tip "💡 결국 다 프롬프트로 통합니다"
    CLAUDE.md 만들기, 명령 저장하기, 스킬·MCP·서브에이전트 확인하기 — 오늘 배운 것 전부, 결국 **우리말로 부탁하는 것**으로 다 되었습니다. 이 책이 계속 말하는 "프롬프트 엔지니어링은 필요 없다"는 말을 오늘도 확인한 것입니다.

!!! warning "⚠️ 조심 — 한 번에 다 외우지 않아도 됩니다"
    `/init`, `/help`, `/mcp`, `/agents` 같은 명령을 지금 다 외울 필요는 없습니다. 잊어버리면 그냥 "지금 쓸 수 있는 명령이 뭐가 있어?"라고 물어보면 됩니다.

    - 넷 다 결국 **우리말 프롬프트**로 다룰 수 있다.
