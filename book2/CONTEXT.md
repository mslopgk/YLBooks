## 확정 결정 (2026-07-08)
1. 기존 25장 pygame 교재(`chapters/`, `book.md`)와 **완전 분리** — 백지에서 새로 집필. 기존은 톤/포맷 참고만. 신규 원고는 `book2/`.
2. 리듬게임은 **Expo(React Native/JavaScript)** 로 제작. 기존 pygame 방식 폐기.
3. **Suno 음악 / Higgsfield 영상 "갈아끼우기" → 제외.** (별개 콘텐츠라 이 책엔 안 넣음.)
4. 바이브코딩 도구 4종 설치 전부 수록:
   - **Claude Code = 주력**(본문 실습 전체를 이걸로 진행).
   - Codex · OpenCode · Antigravity = "설치·첫 실행·같은 프롬프트 넣어보기" 갤러리.
   - 핵심 메시지: **"프롬프트는 어느 도구든 똑같이 통한다."**
5. 완전 왕초보 기초 필수 수록: 터미널, 파일 탐색기, 용어 정리, Node.js 설치.
6. 최신 바이브코딩 요소 총망라: 스킬 · MCP · 서브에이전트. 메시지: **"프롬프트 엔지니어링은 필요 없다."**
7. "코드는 ㄴㄴ" — 독자는 **프롬프트를 입력**하고 AI가 만든 결과를 **실행·확인**만. 소스코드를 읽거나 타이핑하지 않음.
8. 이미지 소싱: (a) 실제로 화면 열어 캡처, (b) 한국인 블로그 이미지, (c) 웹검색. 최대한 많이.
9. 제작 방식: 멀티에이전트. **단순 반복작업만 codex에 위임해 토큰 절약.**

## 해결됨 (2026-07-08 · 예시 프로젝트 `C:\Users\user\keyboard-game\rhythm` 분석 반영)
- **컨트롤러 = 블루투스 무선 · 방향키 4개(← ↓ ↑ →).** 예시 실물 `nordic_keyboard`. USB·D/F/J/K 아님. 페어링만 하면 키보드로 인식.
- **Expo 타깃 = Web(브라우저) 확정.** `npx expo start --web` → 브라우저 `keydown`(Arrow*, keyCode 37/40/38/39)로 방향키 수신. 네이티브는 방향키가 포커스 이동에 가로채여 부적합.
- **노트 자동 채보(오디오 분석 자동 생성) 제외.** 예시의 `audioChart.js`(온셋/BPM 분석) 부분 = 구현 난도 높음 → 책은 **고정 노트 패턴(120BPM 격자)** 만 다룸.
- **키링 펌웨어 불필요.** 게임을 노트북 자판으로 먼저 완성 → 후반부에 키링 블루투스 페어링해 플레이. "코드 ㄴㄴ" 원칙 부합.
- **판정 폭**(예시): PERFECT ±70 / GREAT ±140 / GOOD ±220ms (블루투스 입력 지연 흡수). 지연 보정(±) 기능 있음.
- **조직 단위 = 마당(부) → 차시(lesson) → 단계(스텝).** 확정, `FORMAT.md` 반영.

## 용어집 (초보용 한 줄) — 집필하며 확장
- **바이브코딩** — AI에게 말로 시켜 결과물(프로그램)을 받아내는 방식. 문법 암기 없이 "무엇을 원하는지"만 말한다.
- **터미널** — 컴퓨터에 마우스 대신 글자로 명령을 내리는 창.
- **파일 탐색기** — 내 컴퓨터 안 폴더·파일을 눈으로 열어보는 창.
- **프롬프트** — AI에게 시키는 말(지시문).
- **Claude Code / Codex / OpenCode / Antigravity** — AI에게 시켜 코딩하는 도구(코딩 에이전트) 4종.
- **Expo** — 앱/게임을 만드는 도구(뼈대). 이 책의 리듬게임을 이걸로 만든다.
- **키링 키보드** — USB로 꽂으면 키보드처럼 인식되는 작은 기기. 리듬게임의 컨트롤러.
