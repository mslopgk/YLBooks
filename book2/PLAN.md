## 1. 한 줄 소개 / 가제
컴퓨터가 낯선 사람도 **AI에게 말만 걸어서** 내 리듬게임을 만들고, **키링 키보드**로 직접 플레이하는 책.

**가제 후보**
1. **《말만 하면 게임이 만들어진다 — 바이브코딩으로 만드는 내 키링 리듬게임》** (추천)
2. 《AI한테 시켜서 만드는 첫 게임 — 키링 리듬게임 편》
3. 《코딩 몰라도 됩니다 — 바이브코딩 리듬게임 만들기》

## 2. 대상 독자 / 성공 기준
- **완전 초보.** 컴퓨터를 마우스로만 써 본 60~70대도 혼자 완주 가능한 수준. 전공·영어·수학 전제 0.
- **성공 기준**: 마지막 장을 덮을 때 (1) 브라우저에서 도는 **내 리듬게임 한 편**, (2) **키링 키보드로 실제 플레이**, (3) 남에게 보여줄 수 있는 링크/화면. 코드는 한 줄도 직접 안 썼는데도.

## 3. 이 책의 5가지 원칙 (기존 교재와 다른 점)
1. **코드를 안 보여준다.** 본문 박스는 오직 ①AI에게 넣는 프롬프트 ②화면에 뜨는 안내문. → `.js/.ino/.py` 소스 없음.
2. **프롬프트 위주 + "프롬프트 엔지니어링 필요 없다".** 말하듯 자연스럽게 부탁하면 된다는 걸 매 예시로 증명.
3. **도구 4종을 다 설치**시키되(설치 갤러리), 실습은 **Claude Code 하나**로. "프롬프트는 어디든 똑같이 통한다."
4. **왕초보 기초부터.** 터미널·파일탐색기·용어·Node 설치를 그림으로 처음부터.
5. **그림이 본문보다 먼저.** 한 단계 = 한 그림. 실제 캡처·한국 블로그·웹검색으로 최대한.

## 4. 기술 뼈대 (조사로 확정)
| 항목 | 결정 | 근거 |
|---|---|---|
| 리듬게임 엔진 | **Expo (React Native) — Web 타깃** | 브라우저 `keydown`(Arrow*, keyCode 37/40/38/39)이 방향키를 그대로 수신. 네이티브는 방향키가 포커스 이동에 가로채여 초보 부적합 |
| 게임 실행 | `npx expo start --web` → 브라우저에서 실행 | 앱스토어·설치 전부 생략 |
| 컨트롤러 | 키링 키보드 = **블루투스 무선 · 방향키 4개(← ↓ ↑ →)** | 예시 실물 = `nordic_keyboard`. 페어링만 하면 키보드로 인식, 펌웨어·드라이버 불필요 |
| 노트(채보) | **미리 정해둔 고정 패턴**(120BPM 격자). **자동 채보(오디오 분석) 제외** | 온셋/BPM 분석 자동 생성은 구현 난도 높아 책 범위서 뺌 |
| 주력 도구 | **Claude Code** (`irm https://claude.ai/install.ps1 \| iex`) | 2026 공식 권장 네이티브 설치, Node/WSL 불필요 |
| 갤러리 도구 | Codex(ChatGPT 로그인) · OpenCode(오픈소스, `/connect`) · Antigravity(구글, GUI) | 동일 흐름: 설치→로그인→폴더서 실행→한글 프롬프트 |
| 공통 엔진 | Node.js LTS (nodejs.org) | Claude Code(npm경로)·Expo가 사용 |

> ⚠️ 변동성 큰 값(요금·무료한도·모델명·SDK버전)은 **본문에 못박지 않고** "대략/공식사이트 확인"으로 처리(조사 pitfalls 반영).

## 6. 서사 설계 메모
- **키링은 일찍 "인사"만, 플레이는 나중.** 게임을 (노트북 자판으로) 먼저 완성 → "사실 키링은 특별한 자판일 뿐" → 꽂아서 플레이. "코드 안 짜는" 약속과 맞고, 후반부에 하드웨어로 클라이맥스.
  - 단, 동기부여용으로 **2마당 즈음 "키링 꽂아 메모장에 눌러보기"** 한 컷을 미리 넣어 설렘 유발(정식 플레이는 17~18차시).
- **매 차시 끝 "여기까지 됐으면 ✅ 체크"** 로 성공을 눈으로 확인시켜 이탈 방지.
- **반복 안심 문구**: "잘못 눌러도 컴퓨터 안 망가져요", "5분 막히면 그대로 복사해서 AI에게".

## 7. 이미지 소싱 전략
- `src=capture` — 내가 브라우저/화면 열어 직접 캡처(공식 페이지·게임 화면). **1순위.**
- `src=blog` — 한국 블로그 이미지(설치 마법사 등 OS 장면). **출처 표기 필수**, 저자 최종 확인. (조사에서 Node설치 11컷 urmaru.com, OpenCode/Antigravity elancer 등 소스 확보.)
- `src=web` — 웹검색 이미지(로고·개념).
- `manual` — 실물 촬영(키링 사진·플레이 장면). 저자 몫.
- 관리: `book2/images/cNN/`, 대기열 `CAPTURE_QUEUE.md`.

### OS 화면 캡처 파이프라인(후속)
터미널·파일탐색기·설치 마법사는 브라우저로 못 찍음 → (a) 실기에서 스크립트 캡처(기존 `tools/capture/capture.py` 활용) 또는 (b) 한국 블로그 이미지(출처표기)로 충당. 이번 샘플은 **웹 화면만 실제 캡처**, OS 장면은 대기열로.

## 8. 제작 파이프라인 (멀티에이전트 · 토큰 절약)
- **조사**: 병렬 에이전트(Sonnet) — ✅ 1차 완료(6스레드). 차시별 사실 확인 시 추가.
- **집필**: 품질 핵심이라 **본체(Opus)가 직접**. 톤·안전문구·프롬프트 예시 정교화.
- **단순 반복 → codex 위임**(토큰 절약): FIG 번호 매기기/재정렬, 용어 박스 일괄 삽입, 목차·상호참조 링크 갱신, 부록 D(프롬프트 모음) 자동 수집, 이미지 파일명 정리 등.
- **검증**: 차시 초안마다 "왕초보가 이대로 따라 되나" 리뷰 에이전트 1인 + 사실오류(요금/버전) 체커.

## 9. 리스크 / 진행 전 확인 필요
1. **키링 = 블루투스·방향키 확정**(예시 `nordic_keyboard`). 5마당에 **블루투스 페어링** 절차(그림) 필요. 게임 키는 방향키(← ↓ ↑ →)로 고정.
2. **요금·무료한도·모델명·Expo SDK** 변동 — 본문 못박지 않기로 처리(위 §4 주석).
3. **블로그 이미지 저작권** — 재사용 시 출처표기/허락 또는 저자 재캡처 원칙.
4. **OpenCode에 Claude 계정 연결 경고**(계정 정지 보도) — 9차시 집필 전 공식 공지 재확인, 기본은 도구 자체 provider 사용 안내.
5. **"프롬프트 엔지니어링 불필요" 근거** — 6번 조사 스레드 실패 → 본체 지식으로 집필, 필요시 한국 자료 재조사.
