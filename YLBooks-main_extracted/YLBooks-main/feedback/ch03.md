# ch03 피드백 — Antigravity로 시각적으로 코딩하기

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (가입 진입점·보안 경고 누락으로 fresh 학습자가 첫 단계에서 막힘)

### blocker
1. **Antigravity 신청/다운로드 사이트 진입점 부재** — 본문이 "Antigravity 공식 사이트에서 베타 신청"이라고만 하고 실제 접근할 URL/검색어가 전혀 없음. 처음 쓰는 학습자는 어디로 가야 할지 알 수 없어 가입 자체를 시작하지 못함. → 3.2에 다운로드 진입 안내(공식 사이트 검색어 + 베타 신청 경로) 추가.

### major
2. **설치 순서 역전** — "폴더 열기"(`File → Open Folder`) 설명이 3.6 첫 프로젝트에서야 등장하는데, 그보다 앞선 3.4에서 이미 Manager View(파일 트리 등 4분할)를 설명. 학습자는 폴더를 열어본 적 없는 상태로 파일 트리 화면을 마주함. → 폴더 열기를 Manager View(3.4) **앞** 순서로 재배치.
3. **OS 보안 경고 대처법 누락** — 베타 단계의 서명 안 된 `.exe`/`.dmg`라 윈도우 SmartScreen("알 수 없는 게시자")·Mac Gatekeeper("확인되지 않은 개발자") 경고가 거의 확실히 뜸. 우회 절차가 없어 학습자가 설치를 포기할 위험. → 3.3에 SmartScreen/Gatekeeper 우회 절차 추가.

### minor
- 3.2 경고박스(⚠️)와 바로 다음 본문 단락이 거의 같은 내용 반복 → 본문 중복 정리.
- `Ctrl + 백틱` 단축키에서 "백틱" 키 위치 미설명(Tab 위, 숫자 1 왼쪽) → 한 줄 보충.
- `.gitignore`가 무엇인지 미설명(3.5에서 파일명만 등장) → 한 줄 정의 보충.

### 적용한 보완 (Round 1)
- 3.2에 **다운로드 진입 안내** 추가 — 정확한 URL을 지어내지 않고 "공식 사이트(검색: Antigravity Google IDE)에서 베타 신청" 식으로 안내하고, 확정 주소는 편집 시 채울 자리(`__________`) 표시.
- 3.2 경고박스/본문 중복 정리, `accounts.google.com` 계정 만들기 흐름은 유지.
- 3.3에 **SmartScreen(윈도우)·Gatekeeper(Mac) 보안 경고 우회 절차** 추가.
- "폴더 열기"를 **새 3.4로 분리**해 Manager View(이전 3.4 → 3.5) **앞**으로 재배치. 이후 절 번호·"첫 프로젝트"의 1단계 폴더 열기 문구를 새 흐름에 맞게 정리.
- `Ctrl + 백틱`에 백틱 키 위치 한 줄, `.gitignore` 한 줄 정의 보충.

## Round 2 (2026-06-18) — 베타 승인 대기 흐름 보완

**판정: 콘텐츠 보완 완료** (가입 진입점·보안 경고·폴더 순서는 Round 1에서 해결됨을 재확인. 이번 라운드는 "베타 승인 대기가 학습 흐름을 끊는" 문제를 재프레이밍으로 보완.)

### 재확인 (이미 양호 — 유지)
- 다운로드 진입점: "공식 사이트(검색: Antigravity Google IDE) → 공식 도메인 확인 후 신청 + 승인 메일 링크로 설치" 경로 명확. 정확한 URL은 지어내지 않고 빈칸(`https://_____`) 유지 — 저자/시점 의존이라 본질적 한계.
- SmartScreen(윈도우)·Gatekeeper(Mac) 우회 절차 3.3에 존재 — 유지.
- 3.4 폴더 열기 → 3.5 Manager View 순서 올바름 — 유지. FIG f01~f16 마커·번호·순서 전부 보존.

### 적용한 보완 (Round 2)
- **베타 승인 대기(수 일) 재프레이밍** — 흐름이 끊기지 않도록 세 곳에 일관 적용:
  - 장 상단에 "이 장은 '당일 완주'가 아닐 수 있다 + 대기 중 4·5장 먼저" 안내 박스(🔎) 추가.
  - 3.2 신청 직후에 "승인 메일을 기다리는 동안 할 일" 팁 박스(💡) 추가 — 지금은 Claude Code만으로 4·5장 진행, 메일 오면 3.3으로 복귀. 신청 날짜 메모 빈칸 제공.
  - 3.8 마무리 첫 단락과 연습 문제 1번을 "아직 미승인이면 4·5장 먼저 → 나중에 3.3 복귀" 흐름과 일관되게 수정.

### 도구 베타 한계 (저자/시점 의존 — 콘텐츠로 해결 불가)
- 정확한 공식 다운로드 URL, 실제 승인 소요일, 승인 메일 실제 문구·스크린샷(그림 3.1·3.3·3.5·3.6 등)은 저자가 베타 승인을 받은 시점에만 확정 가능. 콘텐츠로 가능한 보완은 완료했고, 나머지는 도구 베타 특성상 "당일 완주 보장" 불가 — 저자/시점 의존 잔여 항목으로 남김.
