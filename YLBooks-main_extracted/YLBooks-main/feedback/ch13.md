# ch13 피드백 — 캘리브레이션과 지연 보정

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (개념 흐름은 탄탄하나 실행 코드 자기완결성·본문-코드 정합 결함으로 초보가 돌릴 수 없음)

### blocker
1. **`beep_sound` 객체 생성 코드 부재** — `run_calibration(screen, clock, beep_sound)`와 `main()`이 `beep_sound`를 받지만, 이 사운드 객체를 어떻게 만드는지 본문 어디에도 없음. fresh 학습자는 `pygame.mixer.Sound`도, .wav 파일도 준비 못 해 NameError로 즉시 막힘. → 짧은 비프음을 코드로 합성하거나(.wav 파일 안내 포함) `beep_sound`를 정의하는 코드 블록 추가.
2. **pygame/오디오 초기화 부재** — `pygame.init()`, `pygame.mixer.init()`, `set_mode()`(screen), `Clock()`(clock) 호출이 전혀 없음. `main()`은 `screen`/`clock`/`beep_sound`를 정의 없이 사용 → 모두 NameError. → `main()` 진입부에 초기화 + 세 변수 정의 추가.

### major
3. **측정 기준 불일치(본문 vs 코드)** — 13.3 본문·그림 13.4 프롬프트는 "beep **재생 시각**(beat_time) 기준"이라 설명하지만, 코드는 `last_beat_time = next_beat_time`(예약된 시각)을 기준으로 diff를 계산. 실제 `beep_sound.play()` 호출 시각과 예약 시각은 프레임 단위로 어긋날 수 있어 본문 설명과 실제 동작이 불일치. → 실제 `play()` 호출 직후 `time.time()`을 측정해 그 시각을 기준으로 diff 계산하도록 코드·설명 정합화.
4. **flash 깜빡임 1프레임만 유지 → 안 보임** — `flash = True` 직후 같은 루프 끝에서 `flash = False`로 꺼서, 원은 단 1프레임(120fps 기준 ~8ms)만 밝음. 사람 눈에 깜빡임이 거의 안 보임. → 시각(타임스탬프) 기준으로 일정 시간(예 120ms) 동안 밝게 유지하도록 수정.

### minor
5. **offset==0 트리거 빈틈** — `main()`이 `user_offset_ms == 0`일 때만 캘리브레이션을 띄움. 실제 측정 결과가 0.0이면 매 실행마다 다시 캘리브레이션. → `"user_offset_ms" not in settings` 또는 별도 `calibrated` 플래그로 판정하도록 보완.
6. **취소(QUIT) 경로 안내 부재** — `run_calibration`이 중간 종료 시 `None` 반환하지만 본문에 설명 없음. → "도중에 창을 닫으면 None을 반환하고 보정을 건너뛴다" 한 줄 추가.
7. **"다시 하기" 코드 부재** — 13.6에서 '캘리브레이션 다시 하기'를 언급하지만 강제로 다시 돌리는 호출 코드가 없음. → `run_calibration`을 무조건 호출해 덮어쓰는 `recalibrate()` 한 토막 추가.
8. **13.3 블록 불필요 import** — 핵심 코드 상단 `import json`은 이 함수에서 안 씀(저장은 13.4). → 제거.

### 적용한 보완 (Round 1)
- 13.3 핵심 코드 상단 import를 `pygame`/`time`만 남김(`import json` 제거).
- 13.3에 **비프음 만들기** 코드 블록 신설 — `pygame.sndarray`로 880Hz 0.08초 비프를 합성(numpy), .wav 파일이 있으면 `pygame.mixer.Sound("beep.wav")`로 대체 가능함을 안내.
- `run_calibration`을 수정: 비트 발생 시 `beep_sound.play()` **직후** `actual_beat_time = time.time()`을 기록해 그 시각 기준으로 diff 계산(본문·그림 13.4 설명과 정합). flash를 타임스탬프 기반(`flash_until`)으로 바꿔 120ms 동안 원이 밝게 유지되도록 수정. offset==0 트리거 빈틈은 누름 직전 비트가 없을 때 무시하는 로직 유지.
- 13.3 본문에 취소(창 닫기) 시 `None` 반환·보정 건너뜀 설명 한 줄 추가.
- 13.4 `main()`에 pygame/mixer 초기화·`screen`/`clock`/`beep_sound` 정의 추가, 첫 실행 판정을 `"user_offset_ms" not in settings` 기준으로 변경(0.0 측정값 무한 재측정 방지).
- 13.6에 `recalibrate()` 함수 한 토막 추가 — 메뉴의 '다시 하기'가 무조건 `run_calibration`을 돌려 `settings.json`을 덮어쓴다는 흐름 코드로 보강.
- 그림 마커 id·번호·순서(ch13-f01~f11), 아이콘 코너(note/warning/tip/abstract/question/example/info) 전부 유지. 절 번호 변경 없음.

## Round 2 (2026-06-18) — 런타임 정합 재검증

**판정: needs-work → fixed** (Round 1에서 자기완결성은 잡혔으나 믹서 채널 불일치·진입점 부재가 남아 있었음)

### blocker
9. **`make_beep()` 스테레오 배열 ↔ 믹서 채널 불일치** — `make_beep()`는 `np.column_stack`으로 2채널(스테레오) 배열을 만들어 `pygame.sndarray.make_sound`에 넘기는데, 믹서가 그와 다른 채널 수로 초기화되면 `make_sound` 호출 즉시 `ValueError`(array depth/shape mismatch)로 터짐. 13.4의 `main()`은 `pygame.mixer.init()`을 인자 없이 호출해 채널 수를 명시하지 않았음 → 환경/기본값에 따라 비프음 생성이 실패할 수 있음. → `pygame.mixer.pre_init(frequency=44100, size=-16, channels=2)`로 포맷을 못 박아 stereo 배열과 정합화.
10. **`if __name__ == "__main__": main()` 진입점 부재 + `main()`이 캘리브레이션 후 그냥 끝남** — 파일을 직접 실행할 표준 진입점이 없어 fresh 학습자가 무엇을 실행해야 할지 모름. 또 `main()`은 `user_offset_ms`만 구하고 게임 루프(13.5)로 연결되지 않아, 측정만 하고 창이 닫히거나 아무 일도 안 일어남. → 진입점 추가 + 측정 후 결과 화면(`show_offset_result`)을 거쳐 `run_game()`으로 연결.

### 확인 (Round 1에서 이미 처리됨)
- 측정 기준: `beep_sound.play()` 직후 `last_beat_time = time.time()`로 실제 재생 시각을 잡음(line 118). 본문·그림 13.4 설명과 정합 — 추가 수정 불필요.
- flash 가시성: `flash_until = last_beat_time + 0.12`로 120ms 유지(line 120/131). 1프레임 깜빡임 문제 해소 상태 — 추가 수정 불필요.

### 적용한 보완 (Round 2)
- 13.3 비프음 블록 뒤에 **믹서 포맷 정합 설명 + `pygame.mixer.pre_init(frequency=44100, size=-16, channels=2)` 코드 토막** 신설. stereo 2채널 배열과 믹서 채널 수가 같아야 `make_sound`가 동작함을 설명, `make_sound`로 문구 통일.
- 13.3 도입부에 `pip install numpy` 안내 한 줄 추가(numpy 미설치 학습자 대비).
- 13.4 `main()`: `pygame.mixer.init()` → `pygame.mixer.pre_init(...)` + `pygame.init()` 순서로 교체(pre_init은 init 전에 와야 함). 측정 후 `show_offset_result()` + `run_game()` 연결, 끝에 `pygame.quit()` 추가.
- 13.4에 `if __name__ == "__main__": main()` 표준 진입점 추가 + 의미 설명, `show_offset_result()` 한 토막 신설(결과 2초 표시, 그림 13.6과 연결).

## Round 3 (2026-06-18) — grader 미정의 이름 결함 해소

**판정: FAIL → PASS** (`grade_chapter.py 13`가 `main()`의 미정의 호출로 FAIL시키던 결함 제거)

### blocker
11. **`main()`이 `run_game()`을 정의 없이 호출 → NameError** — `grade_chapter.py 13`이 `main()` 블록(완성형)에서 `run_game`을 미정의 이름으로 잡아 FAIL. 본문은 13.5의 게임 루프로 "진입한다"고 했지만 그 게임 루프(`run_game`)는 이 장 어디에도 정의되지 않았고, 12장 함수(`key_to_lane`/`get_nearest_note`/`update_score`)에 의존하는 발췌만 존재했음. → 이 장 단독으로 도는 **보정 확인 루프**(`run_offset_check`)로 대체. 옵션 (b) 채택.

### 적용한 보완 (Round 3)
- 13.4 `main()`의 마지막 호출을 `run_game(screen, clock, user_offset_ms)` → `run_offset_check(screen, clock, beep_sound, user_offset_ms)`로 교체. 본문도 "캘리브레이션→저장→보정 확인까지가 이 장의 핵심이며, 본격 게임 루프 연결은 13.5에서 다룬다"로 정합화. 창 제목도 `"키링 리듬게임 — 캘리브레이션"`으로 조정.
- 13.4에 **`run_offset_check()` 함수 토막 신설** — 1초 비프 박자에 D키를 누르면 누른 시각을 그대로 `judge(press_time, last_beat_time, user_offset_ms)`에 넘겨 PERFECT/GOOD/MISS를 큰 글씨로 표시. ESC/창닫기로 종료. 노트 낙하 게임 없이 보정값 하나만 떼어 검증하는 자기완결 루프. 사용하는 이름(`pygame`/`time`/`judge`/`beep_sound`)은 모두 이 장 안에 정의됨.
- 13.5 게임 루프 발췌 블록 직전에 "**12장 게임 루프 발췌**, 세 함수(`key_to_lane`/`get_nearest_note`/`update_score`)는 12장에서 정의됨" 문구 추가 + 블록 첫 줄 주석도 발췌임을 명시. grader의 fragment 면제가 우연이 아니라 의도된 발췌임을 코드/본문 양쪽에서 분명히 함.
- 흩어진 import(numpy/pygame/time/json/os)는 각 코드 블록 상단에 이미 명시되어 있어(13.3 `import numpy as np`/`import pygame`/`import time`, 13.4 `import json`/`import os`) chapter 전체 union에서 모두 바인딩됨 — `run_offset_check`/`main`이 쓰는 이름에 미정의 없음(확인 스크립트로 검증, 남은 미정의는 13.5 발췌의 12장 함수 3개뿐이며 fragment로 면제).
