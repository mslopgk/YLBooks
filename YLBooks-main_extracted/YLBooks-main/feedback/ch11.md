# ch11 피드백 — 차트 파일과 음악 동기화

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (개념 설명은 좋으나 실행 가능한 통합 코드가 없어 따라 하면 NameError, 음악 파일 준비 절차 모순)

### blocker
1. **동작 확인 가능한 전체 main.py(게임 루프)가 없음** — 11.5/11.6의 코드 조각에 `music_start_tick`, `next_note_time`, `next_note`, `PERFECT_WINDOW`가 정의 없이 등장 → 그대로 따라 치면 `NameError`. 동기화·디버깅을 "실제로" 테스트할 수 있는 통합 코드가 어디에도 없음. → 11.5에 모든 변수가 정의되고 연결된 **실행 가능한 통합 main.py**(차트 로드 → mixer 재생 → 게임 루프 → elapsed_ms/다음 노트 계산 → 디버그 출력)를 추가. `PERFECT_WINDOW`, `next_note`/`next_note_time` 산출 코드 명시.

### major
2. **song.ogg 다운로드 절차 부재 + 파일명 불일치** — 본문/차트는 `assets/song.ogg`인데 저장소 그림(11.8)·설명은 `song_easy.ogg` 등 `assets/songs/`에 있음 → 초보가 어떤 파일을 어디서 받아 무슨 이름으로 두는지 알 수 없음. 또 차트의 `music_file` 필드가 정작 재생 코드(`music.load("assets/song.ogg")` 하드코딩)와 연결되지 않음. → 11.4에 저장소(`github.com/ylbook/keyring-game`)에서 받아 `assets/song.ogg`로 이름 맞추는 구체 절차 추가(URL은 지어내지 않고 저장소 경로만 안내). 재생 코드를 `music.load(chart["music_file"])`로 바꿔 차트와 실제 재생을 연결.

### minor
3. **pygame 설치 미언급** — 11.4에서 `import pygame`을 쓰는데 설치 안내가 없음 → `pip install pygame` 한 줄 추가.
4. **offset 부호 직관 부족** — "늘린다/줄인다"가 코드의 `- chart["offset_ms"]`와 어떻게 연결되는지 불명확 → 식과 방향을 한 문장으로 잇는 설명 보강.
5. **mixer.init() 필요 조건 모호** — "따로 호출해야 하는 경우가 있다"가 애매 → "music.load() 전에 반드시 한 번" 으로 단정.
6. **11.6 "키 누르면 PERFECT/MISS"는 12장(키입력) 선행 모순** — 11장 범위(키입력 없음)에서 확인 불가. → 11장에서 확인 가능한 방식(디버그 출력의 t 값과 노트가 판정선을 지나는 순간을 눈으로 비교)으로 재서술하고, 키 판정은 12장 선행 안내로 명시.

### 적용한 보완 (Round 1)
- 11.4: `pip install pygame` 설치 안내 추가. `mixer.init()` 설명을 "load 전 반드시 한 번"으로 단정. 음악 파일 준비 절차를 신설 — 저장소 `assets/songs/`의 곡(예: `song_easy.ogg`)을 받아 프로젝트 `assets/song.ogg`로 복사·이름 변경하는 단계 명시(차트 `music_file`과 일치시킴). 재생 코드를 `chart["music_file"]` 사용으로 변경.
- 11.5: **실행 가능한 통합 main.py** 추가 — `load_chart` → `mixer.init/load/play` → `music_start_tick` 기록 → 게임 루프에서 `elapsed_ms` 계산 → `PERFECT_WINDOW` 상수 정의 → `next_note`/`next_note_time`(아직 안 지난 첫 노트) 산출 → 디버그 출력. offset 부호와 식의 관계 한 문장 보강.
- 11.6: 디버그 절차를 통합 코드의 출력에 맞춰 정리. "키 누르면 PERFECT/MISS"를 11장 범위(눈으로 t 값과 판정선 통과 시점 비교)로 재서술, 키 판정은 12장 선행 안내로 명시. 어긋남 방향 설명을 통합 코드 변수 기준으로 재정렬.
- 그림 마커·번호·순서(f01~f15), 아이콘 코너(note/tip/warning/abstract/question/example/info) 전부 유지. PROGRESS.md 미변경.

## Round 2 (2026-06-18) — 저자 저장소 의존 제거(자기완결화) + 결함 재점검

**판정: improved** (음악 확보 경로를 외부 저장소 의존에서 학습자 자기완결로 전환, 통합 코드 실측 검증 완료)

### blocker
7. **song.ogg를 존재 여부 불명인 `ylbook/keyring-game` 저장소 다운로드에 의존** — Round 1이 남긴 11.4 "저장소에서 받아 이름 맞추기"는 실재 보장이 없는 URL/폴더(`assets/songs/`, `song_easy.ogg`)에 학습자를 묶어 둠. 그 저장소가 없으면 11장 전체가 실행 불가. → 11.4를 **자기완결 3경로**로 재작성: (1) Pixabay/Freesound/incompetech 등 무료 음원 직접 확보 + Audacity로 ogg 변환, (2) `numpy`+표준 `wave`로 BPM 120 8박 비트를 생성하는 `make_song.py`(추가 인코더 불필요, wav 산출 → music_file만 song.wav로), (3) Suno 등 AI 생성. 모두 "결과물을 `assets/song.ogg`(또는 song.wav)로 두고 차트 music_file과 일치"로 수렴.

### minor
8. **상대경로 실행 주의 부재** — 통합 main.py가 `"assets/..."` 상대경로를 쓰는데 실행 위치 경고가 없었음 → 11.5에 "main.py 폴더에서 실행, 아니면 FileNotFoundError. 앞 장 cd 전제면 OK" 한 줄 추가.
9. **11.6 동기화 확인이 10장 노트 그리기 의존인데 11장 단독 절차가 흐릿** → 확인을 두 갈래로 명시: 11장 단독 = 터미널 t 값과 첫 노트 time(0) 비교, 10장 코드 합친 상태 = 화면 노트 위치 + t 값 병행. 그림 11.12/11.13이 10장 합본 전제임을 명시.
