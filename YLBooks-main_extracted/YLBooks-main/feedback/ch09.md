# ch09 피드백 — pygame 설치와 첫 창 띄우기

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (장 제목이 "설치"인데 정작 Python 설치 절차가 빠져 자기완결성 깨짐)

### blocker
1. **Python 설치 절차 통째 누락** — 장 제목이 "pygame 설치"이고 9.2가 "Python 환경 점검"으로 시작하지만, 정작 Python이 안 깔린 학습자를 위한 설치 안내가 "python.org에서 받으세요" 한 줄뿐. fresh 학습자는 `python --version`이 안 먹히는 순간 막힘. → **python.org에서 받아 설치하는 절차를 Windows "Add Python to PATH" 체크 강조와 함께 추가.**

### major
2. **터미널 여는 법 부재** — 9.2 첫 명령(`python --version`) 전에 "터미널을 열고"라고만 함. fresh 학습자는 터미널이 무엇이고 어떻게 여는지 모름. → **VS Code 터미널 여는 법(Ctrl+`) + Windows 터미널 대안 안내 추가.**
3. **프로젝트 폴더 생성·이동(`cd`) 부재** — 9.2 venv 절에 "프로젝트 폴더로 이동한 뒤"라고만 함. 폴더를 만들고 `cd`로 들어가는 실제 명령이 없어 venv가 엉뚱한 위치(홈/시스템 폴더)에 생김. → **폴더 생성(`mkdir`)·이동(`cd`) 명령을 Windows/Mac 양쪽으로 추가.**

### minor
4. **그림 9.3 캡션 오타** — `(.venv\Scripts\activate` 여는 괄호가 명령의 일부로 들어가 잘못됨(프롬프트 표시 `(.venv)`와 명령 `.venv\Scripts\activate`가 섞임). → 캡션 정리.
5. **VS Code 설치 전제** — 그림 캡션·트러블슈팅이 VS Code를 전제하나 설치 안내가 없음. → 9.2 도입부에 VS Code 권장 한 줄 보강.
6. **main.py 저장 폴더/터미널 위치 안내 부재** — 9.3에서 `python main.py`를 실행하지만 파일을 어느 폴더에 저장하는지, 터미널이 그 폴더에 있어야 한다는 안내가 없음. → 실행 절에 위치 안내 한 줄 보강.

### 적용한 보완 (Round 1)
- 9.2 도입부에 **"개발 도구 준비"** 절 신설: VS Code 권장 + 터미널 여는 법(Ctrl+`) 안내.
- 9.2에 **"Python 설치"** 절 신설: Python 미설치 시 python.org에서 받아 설치, Windows 설치 화면 **"Add Python to PATH" 체크박스 필수** 강조(체크 안 하면 `python` 명령이 안 먹힘). venv 활성화 PowerShell 차단 시 우회(`-ExecutionPolicy`) 한 줄 추가.
- 9.2 venv 절에 **프로젝트 폴더 생성·이동 명령**(`mkdir keyring-game` → `cd keyring-game`) 추가, Windows/Mac 분기.
- 그림 9.3 캡션·지시문 오타 수정: 명령은 `.venv\Scripts\activate`, 프롬프트 표시는 `(.venv)`로 분리.
- 9.3 실행 절에 **main.py를 프로젝트 폴더에 저장 + 터미널이 그 폴더에 있어야 함** 안내 보강.
- 그림 마커 id·번호·순서, 아이콘 코너는 모두 유지(신규 그림 추가 없음 — 기존 9.2 그림 흐름에 텍스트만 보강).

## Round 2 (2026-06-18) — fresh 학습자 재검증

**판정: needs-work** (Round 1로 설치 절차는 자기완결됐으나, "어디서 작업하는가"가 흔들려 연쇄 실패)

### major
1. **VS Code에서 프로젝트 폴더 열기(File → Open Folder) 단계 부재** — 초보 최대 실패 지점. 폴더를 안 열어 둔 채 진행하면 파일 저장 위치·인터프리터·터미널 위치가 제각각이 되어 ModuleNotFound / 인터프리터 불일치 / "main.py를 못 찾음"이 연쇄로 터짐. → **9.2 venv 절에 "keyring-game 폴더를 VS Code로 먼저 열기"를 명시 단계(⚠️ 주의 박스)로 추가**, 9.3 실행 절도 폴더 열기 전제로 저장·터미널 안내 연결.
2. **터미널 종류(PowerShell/CMD/Git Bash) 차이 미설명** — `.venv\Scripts\activate`·실행정책 명령이 PowerShell에서만 그대로 동작하는데 어떤 터미널인지 확인법이 없음. CMD/Git Bash 사용자가 activate에서 막힘. → **"개발 도구 준비"에 터미널 종류 확인법 + 이 책은 PowerShell 기준임을 명시(💡 팁)**, 실행정책 팁을 **PowerShell 전용**으로 명시하고 CMD/Git Bash 대안 한 줄 추가.
3. **터미널 재시작 시 venv 풀림·폴더 이탈 미안내** — 터미널을 닫았다 열면 `(.venv)`가 사라지고 작업 폴더가 홈으로 돌아가는데 복귀 방법이 없음. → **venv 절에 복귀 안내(⚠️ 주의): VS Code로 폴더 열고 Ctrl+` 또는 `cd 절대경로` 후 activate, Windows/Mac 분기.**

### minor
4. **첫 확인 명령이 Windows 단일** — `python --version`만 제시. → Mac은 `python3 --version`도 시도하라는 한 줄 추가.
5. **`import sys` 용도 미설명** — 코드 맨 위 `import sys`가 무엇을 위한 것인지 없음. → 코드 아래 "`sys.exit()`로 깔끔히 종료하기 위함" 한 줄 추가.

### 적용한 보완 (Round 2)
- "개발 도구 준비"에 **터미널 종류(PowerShell/CMD/Git Bash) 확인법 + PowerShell 기준 안내** 팁 추가.
- venv 절에 **VS Code 폴더 열기(File → Open Folder)를 명시 단계**로 추가(저장·터미널·인터프리터가 폴더 기준으로 정렬되는 이점 설명).
- venv 절에 **터미널 재시작 시 복귀 안내**(`cd 절대경로` + activate, Windows/Mac 분기) 추가.
- 실행정책 팁을 **PowerShell 전용**으로 명시 + CMD(`activate.bat`)·Git Bash(`source .../activate`) 대안 추가.
- `python --version` 절에 **Mac `python3 --version`** 한 줄 추가.
- 첫 게임 루프 코드 아래에 **`import sys` 용도** 한 줄 추가.
- 9.3 실행 절을 **폴더 열기 전제**로 다듬음(자동 저장 위치·Ctrl+` 터미널 위치 연결).
- 그림 마커 id·번호·순서, 아이콘 코너 모두 유지(신규 그림 없음 — 텍스트만 보강).
