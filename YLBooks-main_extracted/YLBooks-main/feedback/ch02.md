**판정: needs-work** (권장 경로는 완주 가능하나 초보가 헤맬 빈틈 존재)

### major
1. **윈도우 첫 실행 로그인 인증 단계 누락** — 인증 흐름이 Mac 절(2.5)에만 있음. 윈도우 사용자는 2.4 설치 → 2.7 첫 프롬프트로 가는데 2.7엔 인증 안내가 없어 첫 실행에서 당황. → **인증을 OS 공통 단계로 2.7에 올린다.**
2. **터미널 여는 법 부재** — 2.3에서 "터미널을 열고"라고만 함. 여는 방법은 2.4(윈도우)에서야 등장. **Mac Terminal 여는 법은 전무.** → 2.3 앞에 OS별 터미널 여는 법 보강.
3. **Homebrew 미설명/미설치 안내** — "Homebrew가 있다면"인데 무엇인지·설치법 없음. → Mac 절에 Homebrew 한 줄 정의 + 설치 명령 보강.
4. 권한 모드 선택지(Ask each time/Always allow) 문구의 실제 화면 대조 어려움 — minor~major.

### minor
- `cd`·경로 자리표시자(`%USERPROFILE%`, `~`) 미설명 (2.7)
- winget 미설치 구버전 윈도우 분기 없음(방법1로 우회 가능)
- 명령 결과/첫 응답 텍스트 예시 부재(그림은 placeholder 상태)
- Node 설치 후 새 터미널 필요 안내가 PATH 일반론(2.6)에서야 등장

### 적용한 보완 (Round 1)
- 2.3에 "터미널 여는 법"(윈도우 PowerShell / Mac Terminal) 박스 추가
- 2.5/2.3에 Homebrew 정의 + 설치 안내 추가
- 2.7에 **OS 공통 첫 실행 로그인 인증** 단계 추가 + `cd`·경로 자리표시자 한 줄 설명

## Round 2 (2026-06-18) — fresh 학습자 재검증

**판정: needs-work** (권장 경로 완주 가능, 첫 실행 인증에서 초보 혼란 잔존)

### major
1. **계정 종류 선택 단계 누락** — Claude Code 첫 실행 로그인 메뉴에서 "Claude 구독 계정(Claude.ai)" vs "Anthropic Console(API 키)"를 먼저 고르는데 안내가 없어, 초보가 API 키 쪽을 골라 막힘. → 2.7 인증에 **"구독(Claude.ai) 계정으로 로그인" 선택 명시.**
2. **인증 설명 분산** — Round 1에서 2.7에 OS 공통 인증을 넣었으나 2.5에도 "첫 실행 인증 흐름" 프로즈가 남아 이중 설명. → 2.5는 "2.7로 모았다"는 포인터로 축소, 인증 본문은 2.7 한곳으로 통합. 토큰 복사·붙여넣기 손동작(PowerShell 우클릭/Ctrl+V, Mac Cmd+V) 추가.
3. **Node 설치 직후 새 터미널 안내가 PATH절에만** — 2.3 Node 설치 단계 자체에 "새 터미널 열기" 안내 없음. → 2.3에 ⚠️ 박스 + 본문에 "새 터미널 창" 명시.

- 2.2 제목 "가입과 Pro 플랜 결제"가 결제 필수처럼 읽힘 → "가입하고 플랜 고르기 (Free로 시작 OK)"로 톤 일치
- UAC 팝업/ExecutionPolicy 우회 한 줄 안내 부재 → 2.4 방법1에 한 줄 추가

## Round 3 (2026-06-18) — 절차 정확성 점검

**판정: needs-work → 수정 완료** (권장 경로 완주 가능, 잔여 결함 3건 수정)

### major
1. **2.6 PATH 절이 첫 실행(2.7) 앞에 무조건 놓임** — 책 구조상 PATH 절(2.6)이 첫 실행(2.7)보다 앞에 오는데, PATH 문제가 없는 다수 독자도 이 절을 정상 단계로 오해해 불필요하게 환경변수를 만짐. → 2.6 도입부에 역방향 안내(`claude --version` 정상이면 건너뛰고 2.7로) 추가.
2. **첫 대화 종료를 `Ctrl+C`로 안내 (부정확)** — 실제 Claude Code에서 `Ctrl+C`는 '입력 줄 취소'이고 종료하려면 빈 프롬프트에서 두 번 눌러야 함. 정확한 종료는 `/exit`. → `/exit` 우선, `Ctrl+C`는 동작 원리(취소/두 번) 설명과 함께 보조 수단으로 격하.

### minor
- 인증을 빈 폴더 안에서 꼭 해야 하는지 모호(2.7 준비 단계가 빈 폴더 생성 직후라 결합돼 읽힘) → "인증은 폴더 무관, 처음 한 번"임을 2.7 인증 도입부에 한 줄 명시.

### 적용한 보완 (Round 3)
- 2.6 도입부에 `💡 팁 — 이미 잘 되면 건너뛰세요` 박스 추가(정상 시 2.7로 직행 안내)
- 2.7 대화 종료 안내를 `/exit` 우선 + `Ctrl+C` 동작 원리/보조 격하로 정정
- 2.7 인증 도입부에 "폴더 무관·최초 1회" 한 줄 추가
- FIG 마커 17개(f01~f17)·번호·순서·admonition 규약 전부 유지 확인 (수정은 본문 텍스트만)
