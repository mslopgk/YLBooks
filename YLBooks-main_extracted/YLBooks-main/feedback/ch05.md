# ch05 피드백 — arduino-cli 설정과 첫 업로드

## Round 2 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (Windows/PowerShell 전제인데 bash 문법이 섞여 명령이 실제로 동작하지 않는 자리들)

### major
1. **`grep`이 PowerShell에 없음** — 5.3/5.4의 `arduino-cli board listall | grep -i micro`는 macOS·Linux 명령. Windows PowerShell에는 `grep`이 없어 그대로 치면 실패. → PowerShell `| Select-String micro`(또는 전체 목록 보기) 병기.
2. **compile&&upload "한 줄로 묶기" 팁이 bash 전용** — `&&` + 줄 끝 `\`는 bash 문법. PowerShell 5.1은 `&&` 미지원, 줄 잇기는 백틱. → 기본은 "두 줄로 차례대로", 한 줄 버전은 셸별(bash / PowerShell 7+) 대안 명시.
3. **Blink 스케치 폴더/파일을 어디에 어떻게 만드는지 불명확** — 어느 위치에 `mkdir`로 만들고, compile은 그 부모 폴더에서 실행해야 함이 빠짐. → 작업 위치 가정 + `mkdir Blink` + `.ino`는 폴더 바로 안 + compile은 부모 폴더에서 실행 명시.

### minor
- **Pro Micro 내장 LED 단정** — 5.7이 "깜빡인다"로 단정하나 Pro Micro 계열은 내장 LED가 없거나 RX/TX라 안 보일 수 있음 → "보드에 따라 안 보일 수 있다, 업로드 성공이면 정상" 한 줄.
- **board list FQBN/Board Name 빈 칸** — Pro Micro는 board list에서 보드명·FQBN이 비어 나오는 경우 많음 → 포트만 보이면 충분, `arduino:avr:micro` 직접 적어 진행 가능 안내.
- **config init 동작 미설명** — 무엇을 하는지 한 줄(설정 파일 생성, 최초 1회, "이미 있다" 메시지는 무시 가능) 보강.

### 적용한 보완 (Round 2)
- 5.3: `board listall` 필터를 OS별(grep / Select-String / 전체 목록)로 분기 표기.
- 5.4: 본문 내 `grep` 참조 제거하고 `Select-String micro`로 정정.
- 5.6: "한 줄로 묶기" 팁을 "두 명령 이어서 실행" + 두 줄 기본 + bash/PowerShell 7+ 한 줄 대안으로 재작성. 연습문제 3도 `&&` 강제 표현 완화.
- 5.5: 작업 위치 가정 + `mkdir Blink` + `.ino` 위치 + compile은 부모 폴더 실행 안내 추가. 5.6 compile 절에도 부모 폴더 실행 한 줄.
- 5.3: `config init` 동작/최초 1회/중복 메시지 무시 한 줄 추가.
- 5.4: Pro Micro board list 빈 칸이어도 포트만 있으면 진행 가능 안내.
