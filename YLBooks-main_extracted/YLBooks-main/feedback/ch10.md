# ch10 피드백 — 노트가 떨어지는 화면 만들기

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (개념·조각 코드는 충실하나 통합 전체 코드 부재로 초보가 조립 불가)

### major
1. **통합 전체 main.py 미제시** — 코드가 레인 그리기, Note 클래스, time 측정, draw_note, RAW_NOTES, 게임 루프 등 6개 조각으로 흩어져 있고 한 번도 합쳐진 전체 파일이 등장하지 않음. fresh 학습자는 import/클래스/함수/루프의 배치 순서를 알 수 없어 조립 실패. → 장 끝(10.7 직전, 새 절 10.7로 추가하고 예고를 10.8로 이동)에 "지금까지 합친 전체 코드" 블록 신설. import → 상수 → Note 클래스 → draw 함수 → 노트 리스트 → 게임 루프 순서 확정.
2. **`import time` 누락** — `time.perf_counter()`를 10.4·10.5에서 사용하지만 헤더 import 블록(10.2)에는 `import pygame`, `import sys`만 있음. → 통합 전체 코드 상단에 `import time` 포함.
3. **`while True:` 루프 2회 제시로 최종 모호** — 10.2의 레인만 그리는 루프와 10.5의 노트까지 그리는 루프 두 개가 있어 어느 것이 최종인지 불명확. → 10.5 루프에 "앞(10.2)의 루프를 이걸로 교체" 명시, 통합 전체 코드로 최종본 확정.

### minor
- **`field` 미사용 import** — 10.3의 `from dataclasses import dataclass, field`에서 `field`는 어디서도 안 씀. → `from dataclasses import dataclass`로 축소.

### 적용한 보완 (Round 1)
- 10.3 import를 `from dataclasses import dataclass`로 수정(`field` 제거).
- 10.5 노트 루프 도입부에 "10.2에서 작성한 `while True:` 루프를 아래 루프로 통째로 교체하세요" 안내 추가.
- 새 절 **10.7 지금까지 합친 전체 코드** 추가 — `import`(pygame/sys/time) → 상수 → `draw_lanes` → `Note` 클래스 → `draw_note` → `RAW_NOTES`/`notes` → 시작 시각 → 게임 루프(노트 제거 포함) 순서로 완결된 main.py 한 벌 제공. 기존 그림 10.16(코드 구조 정리)을 이 절 안으로 자연스럽게 배치.
- 기존 "10.7 다음 장 예고"를 **10.8**로 번호 이동. 그림 마커 id·번호·순서, 아이콘 코너(note/warning/tip/abstract/question/example/info) 전부 유지.
