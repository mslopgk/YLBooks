# ch06 피드백 — 버튼 하나로 키보드 흉내내기

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (권장 경로는 완주 가능하나 안전·복구 절차에 모순·미정의 존재)

### major
1. **안전 모드 완성 코드 부재 + 방식 모순** — "안전 모드(SAFETY_MODE)를 꼭 습관화하라"면서 그게 통합된 완성 코드를 안 보여줌. 게다가 먼저 보여준 D10 하드웨어 토글핀 방식은 "PCB 완제품에선 외부 점퍼선을 못 꽂는다"고 직후에 부정 → 모순. → D10 방식을 제거(완제품에 무용)하고, `SAFETY_MODE` 소프트웨어 방식의 **완성 코드(6.5 button_a에 통합된 형태)**를 제공. 6.5 메인 스케치 자체에 `SAFETY_MODE`를 처음부터 포함.
2. **monitor 안내와 코드 불일치** — `arduino-cli monitor`로 "코드 흐름 확인"하라는데 제공 코드에 `Serial.begin`/`Serial.print`가 전혀 없어 모니터가 빈 화면. → 6.5 완성 코드에 `Serial.begin(9600)` + 안전모드/버튼 상태 `Serial.print` 디버그 출력을 추가해 모니터에 실제로 글자가 흐르도록 함.

### minor
- `empty` 빈 스케치 **생성 절차** 부재(폴더+파일 만드는 한 단계가 6.7에만 암묵적) → 6.7에 폴더/파일 생성 한 줄 명시.
- **더블탭 후 포트 확인 모순** — "무한 입력 중엔 타이핑조차 어렵다"면서 `board list`로 포트 확인하라고 함(board list는 타이핑 필요). → 미리 평소 포트를 적어두고 그 번호로 바로 upload, 모르면 board list는 차선책으로 격하.
- **COM3 하드코딩** — "자기 포트로 바꾸라"가 6.6에 1회뿐. 6.7 복구 명령에도 동일 주의 1줄 추가.

### 적용한 보완 (Round 1)
- 6.5 완성 코드를 `SAFETY_MODE`(소프트웨어 안전 모드) + `Serial` 디버그 출력이 처음부터 통합된 button_a.ino 전체로 교체. 검토 포인트에 안전모드/시리얼 항목 추가.
- 6.7에서 D10 하드웨어 토글핀 코드 블록 삭제. "사전 예방"을 6.5에 이미 넣은 `SAFETY_MODE` 사용·검증 흐름(시리얼 모니터로 확인 후 false 전환)으로 재서술.
- 6.7 복구 절차: `empty` 폴더/파일 생성 단계 명시, 더블탭 직후 미리 적어둔 평소 포트로 upload(board list는 차선책), COM3 포트 교체 주의 추가.
- 그림 마커·번호·순서, 아이콘 코너 전부 유지. 그림 6.14 캡션은 동일(SAFETY_MODE 강조)하게 유지.

## Round 2 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (권장 경로는 완주 가능하나 ATmega32U4 USB 재열거에 따른 포트 변경 미안내, 복구 실전 명령 불완전)

### major
1. **업로드 후 COM 포트 번호 변경 미안내** — Pro Micro/Micro(ATmega32U4)는 업로드 직후 USB 재열거로 COM 번호가 바뀌는 일이 잦은데, 6.6 시리얼 모니터/재업로드 단계에서 "방금 COM3로 올렸으니 COM3로 모니터"라고만 안내 → 포트를 못 찾으면 학습자가 막힘. → 6.6 4단계에 "업로드 후 포트 번호가 바뀔 수 있으니 `board list`로 다시 확인" 경고 박스 추가, 5단계에도 동일 주의 1줄 추가.
2. **무한입력 복구 시 부트로더 포트 변경 후 실전 명령 불완전** — 더블탭 후 부트로더 포트가 평소와 달라질 수 있다는 언급만 있고, 그 새 포트로 어떻게 다시 upload하는지 구체 흐름이 없음. → 6.7에 "포트 못 찾으면" 별도 단계(5)로 분리: 부트로더 상태에서 `board list`로 새 번호 확인 → 더블탭 다시 → 그 번호로 upload하는 구체 명령(예: COM7) 흐름 보강.

### minor
- **SAFETY_MODE=false 블록에 Serial 디버그 없음** — 완성 코드의 Serial 출력이 안전모드(true) 블록에만 있어, 해제 후 모니터를 열면 빈 화면 → 모니터 안내와 불일치. → false 분기에도 `[LIVE] button = ...` 최소 출력 + `delay(20)` 추가. 6.6 5단계에 "해제해도 [LIVE] 출력이 흐른다" 안내 일치.

### 적용한 보완 (Round 2)
- 6.6 4단계: "업로드 후 COM 포트 번호가 바뀔 수 있다" warning 박스(+`board list`) 추가.
- 6.6 5단계: 재업로드 시 포트 변경 주의 + 해제 후 `[LIVE]` 시리얼 출력 확인 안내 추가.
- 6.5 완성 코드 loop(): SAFETY_MODE=false 분기에 `[LIVE]` Serial 출력 + `delay(20)` 추가.
- 6.7 복구: 부트로더 포트 변경 시 `board list`로 새 번호 확인 → 그 번호로 upload하는 구체 5단계 흐름으로 보강.
- 그림 마커/번호/순서, 아이콘 코너 전부 유지. 신규 그림 추가 없음(기존 6.1~6.15 그대로).
