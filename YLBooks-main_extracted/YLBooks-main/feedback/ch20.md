# ch20 피드백 — 키링 케이스와 마감

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (개념·흐름은 좋으나 파일 받는 절차가 없고, 레진 옵션이 "비워 두라"고만 하고 방법이 없어 그대로 따라 하면 키링이 묻혀 파손)

### blocker
1. **깃허브에서 파일(도면 PDF, STL) 받는 방법이 본문에 없음** — 20.3은 "`templates/paper-case.pdf`를 받습니다", 20.4는 "저장소에 `templates/case.stl`이 있습니다"라고만 함. 초보가 저장소를 열어 폴더를 따라 들어가 다운로드하는 구체 단계가 한 줄도 없음. → 11장의 다운로드 절차 패턴을 재사용해 단계(저장소 열기 → 폴더 이동 → Download / Download ZIP)로 명시. URL은 책 전체가 쓰는 `https://github.com/ylbook/keyring-game`만 사용(지어내지 않음).
2. **옵션 C(레진): "USB-C 포트·버튼 위치를 비워 두라"는 경고만 있고 방법이 없음** — 그대로 부으면 포트·버튼이 레진에 묻혀 키링이 파손/작동 불가. 작업 흐름(20.5)에도 비우는 단계가 없음. → 포트·버튼을 마스킹테이프로 감싸고, 몰드에서 그 부분을 점토/블루택으로 막아 레진이 닿지 않게 하는 구체 절차 추가. 작업 흐름과 정합화하고, 보안경 착용을 흐름 1단계로 편입.

### major
3. **레진 케이스 고리 구멍 만드는 법 부재** — 20.6은 "종이는 송곳, 3D는 모델링됨"만 언급. 레진은 굳으면 못 뚫음 → 경화 전에 몰드 끝에 이쑤시개/스트로 조각을 세워 구멍 자리를 확보하는 법 추가.
4. **100% 스케일 실제 인쇄 설정 안내 부족** — "100% 스케일로 출력"만으로 초보는 프린터 대화상자의 "페이지 맞춤(Fit)"을 못 끔 → "페이지 맞춤 끄기 / 실제 크기(100%) 선택 / 자로 검증" 절차 추가.
5. **종이 박스 측면 높이가 PCB 두께를 못 담는 문제 미대응** — 측면이 너무 낮으면 PCB·부품이 안 들어감 → 측면 높이를 PCB+부품 두께에 맞춰 조절(필요시 측면 덧대기)하는 안내 추가.

### minor
6. **사포 "방(번수)" 용어** — "400~800 방"이 초보에게 생소 → "방=사포 거칠기 번수, 숫자 클수록 곱다"는 한 줄 설명 추가.
7. **슬라이서 설명 후 "안 해도 됨" 혼선** — 슬라이서/G-code를 길게 설명한 뒤 "직접 할 필요 없다"고 해 김 빠짐 → "관리자에게 STL만 주면 됨. 슬라이서는 그쪽이 쓰는 도구라 참고만"으로 정리해 혼선 제거.
8. **레진 "적당량" 모호** — 몰드를 채우는 기준 없음 → "몰드 깊이의 절반쯤 → PCB 넣고 → 부품이 잠길 만큼 보충"으로 단계화.

### 적용한 보완 (Round 1)
- 20.3: 도면 PDF 다운로드 절차를 단계(저장소 열기 → `templates` 폴더 → `paper-case.pdf` Download / 또는 Download ZIP)로 신설. 100% 실제 크기 인쇄 절차("페이지 맞춤" 끄기·실제 크기 선택·자로 검증) 추가. 측면 높이를 PCB+부품 두께에 맞춰 조절하는 안내를 접기 순서에 통합.
- 20.4: STL 다운로드 절차를 동일 패턴으로 신설. 슬라이서 설명을 "관리자에게 STL만 전달, 슬라이서는 그쪽 도구" 로 정리해 "안 해도 됨" 혼선 제거. 사포 "방(번수)" 한 줄 설명 추가.
- 20.5: 안전(보안경 포함) 착용을 작업 흐름 1단계로 편입. 포트·버튼 마스킹(마스킹테이프+몰드 막기) 단계를 흐름에 추가하고 본문 경고와 정합화. "적당량"을 단계화(절반→PCB→보충). 레진 고리 구멍(경화 전 이쑤시개/스트로로 자리 확보) 절차 추가.
- 20.6: 레진 케이스 고리 달기를 "20.5에서 확보한 구멍 사용"으로 연결.
- 그림 마커·번호·순서(f01~f20), 아이콘 코너(note/tip/warning/abstract/question/example/info) 전부 유지. PROGRESS.md 미변경.

## Round 2 (2026-06-18) — 자기완결 전환 + 결함 보강

**판정: 보강 적용** (종이 케이스가 외부 PDF 다운로드에 의존해 자기완결성이 깨지던 문제 해결 + 레진 포트/버튼 노출 검증 단계 보강)

### blocker
9. **종이 케이스가 ylbook 저장소 PDF 다운로드에 100% 의존** — 인터넷·프린터가 없으면 따라 할 수 없어 자기완결성이 깨짐. 도면은 단순 직육면체라 치수만 있으면 자·연필로 그릴 수 있는데 본문에 치수가 한 줄도 없었음. → 밑면·뚜껑·긴/짧은 측면 가로·세로·높이(mm) 치수를 **본문 표로 인라인**하고, 자로 보드 실측 → 표대로 전개도 직접 그리기를 기본 경로로 신설. PDF 출력은 "보조 방법" 팁으로 강등. (STL은 외부 유지 — 3D 모델은 종이처럼 손으로 못 옮기므로 정당.)

### major
10. **레진: 마스킹 후 굳은 레진에서 포트·버튼을 다시 노출/작동시키는 단계 없음** — 기존 흐름은 9단계 "테이프 떼고 추가 경화"에서 끝나, 테이프 가장자리로 새어든 얇은 레진 막이 포트·버튼을 덮어도 잡아내지 못함. → 10단계 신설: 얇은 막 긁어 제거(말랑할 때) + 케이블 실제 꽂아 충전 확인 + 버튼 4개 끝까지 눌림 확인 + 걸리면 둘레 깎기. 20.5 요약 문장도 "노출·작동 확인"까지 포함하게 수정.

### 적용한 보완 (Round 2)
- 20.3: 도입을 "A4 종이+자·연필" 자기완결로 전환. 치수 표(밑면 62×42 / 뚜껑 64×44 / 긴 측면 62×12 ×2 / 짧은 측면 42×12 ×2, 풀칠 날개 7mm) 인라인. 보드 실측 안내 + 전개도 그리는 5단계 + USB-C 구멍 위치 잡기 추가. PDF 출력은 별도 팁(💡)으로 강등하고 "페이지 맞춤 끄기·100%·자로 검증"은 보조 단계로 유지. f03 캡션·type 스크린샷→일러스트(전개도 도면)로 정합화(images.yaml 동기).
- 20.4(영향): 자르기 본문을 "그린(또는 출력한) 도면" 표현으로 일반화. f04 캡션 동기.
- 20.3 접기/USB 문단: "도면 측면 높이"→"표의 측면 세로 12mm", "도면 USB 구멍"→"측면에 그려 둔 구멍"으로 치수 표와 정합화. 측면 부족 시 치수 다시 그리기/덧대기 안내 유지.
- 20.5: 10단계(포트·버튼 노출·작동 확인) 신설. 요약 문장 보강.
- 연습문제 1: "보드 실측→표 보고 전개도 직접 그리기"로 교체(PDF는 선택).
- 핵심 요약: "보드 자로 재서 치수 표대로 직접 그리면 프린터 없이 가능" 추가.
- 그림 마커·번호·순서(f01~f20) 유지(f03만 type 조정, 번호·위치 불변). 아이콘 코너 전부 유지. PROGRESS.md 미변경.

## Round 3 (2026-06-18) — 종이 케이스 기하 결함 수정

**판정: 보강 적용** (치수 표대로 접으면 상자가 안 닫히던 진짜 결함 수정 + 뚜껑 고정법 추가로 체크리스트 성립)

### blocker
11. **측면-측면 결합 수단 부재로 상자가 안 닫힘** — 밑면 네 변에 측면 4장만 그려 접으면 이웃한 측면끼리 옆(세로) 모서리에서 서로 떨어져 있어 네 모서리가 벌어짐. 기존 "측면 바깥 끝 7mm 풀칠 날개"는 측면을 본체에 붙이는 게 아니라(이미 밑면과 한 장으로 이어져 있음) 모서리 결합과 무관한 위치라 닫힘에 기여하지 못함. → **짧은 측면 2장의 좌·우 양 끝(세로 모서리)에 결합 날개(접착 탭) 4장(폭 10mm × 세로 12mm)** 을 두어, 안쪽으로 접어 긴 측면 안쪽 면에 풀로 붙여 네 모서리를 닫는 방식으로 교체. 치수 표·전개도 그리는 순서(2→3단계)·접는 순서·f03 일러스트 지시를 모두 이 방식에 맞춰 정합화.

### major
12. **뚜껑(64×44) 본체 고정법 부재 → 체크리스트 "단단히 닫혀 있는가" 미성립** — 뚜껑을 따로 그리기만 하고 본체에 어떻게 고정하는지 안내가 없었음. → 끼움(뚜껑 변에 7mm 날개 접어 본체 바깥에 끼움)/테이프(한 변 마스킹테이프 경첩)/풀(한 변 고정) 세 방법을 💡 팁으로 추가. PCB를 꺼내야 하므로 여닫이 가능 방식 권장.

### 적용한 보완 (Round 3)
- 20.3 치수 표: "결합 날개 ×4 (10 × 12, 짧은 측면 양 끝)" 행 추가. 측면만으로는 안 닫히는 이유를 ⚠️ 주의 코너로 명시.
- 20.3 전개도 그리는 순서: 기존 3단계 "측면 바깥 끝 7mm 풀칠 날개"를 "짧은 측면 양 끝 결합 날개 4장(접는 점선+빗금)"으로 교체. 측면–날개 점선/풀칠 면 표기 일치.
- 20.3 접는 순서: 측면 세우기 → 결합 날개 안쪽 접기 → 긴 측면 안쪽에 풀칠 → 모서리 고정 → 구멍 다듬기 순으로 재작성(2~5단계).
- 20.3 뚜껑 고정 💡 팁(끼움/테이프/풀) 신설 → 체크리스트 "단단히 닫혀 있는가" 성립.
- f03 일러스트 지시문에 결합 날개 4장·치수(10mm)·모서리 결합 화살표 반영(번호·위치·type 불변).
- 그림 마커·번호·순서(f01~f20) 전부 유지. 아이콘 코너 전부 유지. PROGRESS.md 미변경. FORMAT_SPEC 규약 준수.

## Round 4 (2026-06-18) — 종이 뚜껑 버튼·조이스틱 개구부 누락 수정

**판정: 보강 적용** (뚜껑에 입력부 개구부가 없어 닫으면 버튼·조이스틱을 쓸 수 없던 모순 해소)

### blocker
13. **종이 케이스 뚜껑에 버튼·조이스틱 개구부의 위치·치수가 도면/표에 전혀 없음** — 뚜껑(64×44)을 통판으로만 그리고 덮으니, 닫은 상태에서는 버튼을 누르거나 조이스틱을 움직일 수 없음. 그런데 자르기 7단계 "버튼 위치 다듬기"와 20.7 체크리스트("4개 버튼이 눌리는가 / 조이스틱 4방향")는 **닫은 채 작동**을 요구 → 정면 모순. (레진 옵션은 10단계에서 이미 노출·작동 확인을 다루므로 종이 옵션만 보완.) → 치수 표에 "뚜껑 버튼 구멍 ×4 (8×8mm)", "뚜껑 조이스틱 구멍 ×1 (14×14mm, 있는 보드만)" 행 추가. 버튼·조이스틱이 보드 어디에 있는지(4장 핀맵 D2~D5 기준)와 뚜껑에서 그 자리를 보드로 떠서(종이 덮어 누르기/자로 거리 재기) 잡는 법을 한 단락으로 신설. 그리는 순서 4단계에 뚜껑 구멍 표시, 자르기 순서에 뚜껑 구멍 오려 내기(5단계)+덮어 맞춤 확인(6단계) 추가. ⚠️ 주의 코너로 "구멍 없으면 닫고 못 누름" 명시.

### 적용한 보완 (Round 4)
- 20.3 치수 표: "뚜껑 버튼 구멍 ×4 (8×8)", "뚜껑 조이스틱 구멍 ×1 (14×14, 조이스틱 보드만)" 행 추가.
- 20.3 신설 단락: 버튼 4개(D2~D5)는 보드 윗면에 모여 있고 조이스틱은 옆/한쪽 끝 — USB-C 쪽을 "아래"로 방향 고정 후, 뚜껑 종이를 보드에 덮어 부품 머리 자국으로 위치 떠서 점 찍기 → 버튼 8mm·조이스틱 14mm 네모. 위치는 내 보드로, 책 치수는 구멍 크기 기준값.
- 20.3 그리는 순서 4단계: 뚜껑에 버튼 4구멍·조이스틱 구멍 실선 표시.
- 20.3 자르기 순서: 5단계(뚜껑 구멍 X자→커터칼로 오려 내기 + USB-C 구멍 다듬기), 6단계(덮어 보고 부품 노출 확인, 어긋나면 1~2mm 넓히기/재오림) 추가. 기존 5단계(USB·버튼 다듬기) 흡수.
- ⚠️ 주의 코너 신설: 통판 뚜껑이면 닫고 못 누름 → 7단계·체크리스트 모순. 반드시 구멍 뚫기.
- f03 일러스트 지시문에 뚜껑 버튼 구멍 4개(8×8)·조이스틱 구멍(14×14) 실선 표기 반영(번호·위치·type·caption 불변, images.yaml 동기 불필요).
- 그림 마커·번호·순서(f01~f20) 전부 유지. 아이콘 코너 전부 유지. 친근체 유지. PROGRESS.md 미변경. FORMAT_SPEC 규약 준수.
