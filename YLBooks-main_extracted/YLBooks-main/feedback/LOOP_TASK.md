# 교재 자기개선 루프 — 1회 이터레이션 절차

목표: **메모리·사전지식 없는 새 학습자(fresh agent)가 강의자료만 보고 실제로 끝까지 만들 수 있는가?** 를 단원별로 검증하고, 부족분을 교재에 보완한다. 모든 단원이 "사전지식 없이 완주 가능" 판정을 받으면 루프 종료.

## 방법론 원칙 (drift 방지 — 매 이터레이션 준수)
- **이중 게이트**: 한 챕터가 `satisfied`가 되려면 ① `python tools/grade_chapter.py N` 가 PASS(결정적·구조/완성코드) **그리고** ② fresh-LLM 학습자가 satisfied(의미적). 둘 중 하나라도 실패면 needs-work.
- **fresh 격리 컨텍스트**: 평가·보완 모두 챕터당 독립 서브에이전트(컨텍스트 격리)로 → 세션 drift 차단.
- **fan-out 상한 ≤ 10**: 한 번에 띄우는 서브에이전트는 최대 10개. 더 많으면 배치로 나눈다.
- **front-load**: 각 서브에이전트 프롬프트 맨 위에 핵심 제약(FORMAT_SPEC 그림마커·번호 규약, 자기완결성, 친근체)을 먼저 박는다. (컴팩션/요약에도 살아남게)
- **골든 스펙**: `FORMAT_SPEC.md`가 형식 골든, grader가 그 결정적 집행자.

## 매 이터레이션 절차

1. **대상 선정**: `feedback/PROGRESS.md`를 읽어 다음 평가할 챕터를 고른다.
   - 우선순위: `status=needs-work` > `status=pending`(미평가) > round 수 적은 것.
   - 모든 챕터가 `status=satisfied`면 → **루프 종료**(더 이상 스케줄하지 않음). 종료 사실을 PROGRESS.md 상단에 기록.

2. **Fresh 평가자 띄우기** (general-purpose 서브에이전트, 프로젝트 맥락 차단):
   - **오직 `chapters/chNN.md` 하나만** 읽게 한다. FORMAT_SPEC·다른 챕터·MEMORY·images.yaml 등 **다른 파일 열람 금지**.
   - 역할(중요·올바른 기준): "이 책을 **1장부터 순서대로 따라온** 비전공 학습자. 앞 장들의 결과물(설치한 도구 — Claude Code·arduino-cli+avr코어·Python+pygame·VS Code; 앞 장에서 만든 코드; 익힌 개념 — 터미널/`cd`/스케치 폴더 규칙 등)은 **이미 갖추고 있다.**"
   - **결함으로 칠 것만**: (a) 이 챕터 코드가 실제로 안 돎(미정의 이름·구문오류·논리버그), (b) 이 챕터가 새로 요구하는데 본문에도 앞 장에도 없는 단계/지식, (c) 본문 내부 모순·틀린 서술, (d) 진짜 외부검색이 필요한 미설명.
   - **결함 아님(깎지 말 것)**: 앞 장이 이미 가르친 것(arduino-cli 설치·VS Code·터미널·`cd`·pygame 설치 등)이 "이 챕터에 다시 없다"는 점. 교재는 순서대로 쌓인다.
   - 임무: 본문을 **그대로 따라** 해당 챕터의 결과물을 만들 수 있는지 검증. 다음을 구조화 보고:
     - **완주 가능 여부**(예/아니오)와 막힌 지점
     - **외부 검색/사전지식이 필요한 부분**(본문만으로 불충분한 곳)
     - **누락·모호·순서 오류·전제 누락**(설치 전 사용, 미정의 용어, 코드 불완전 등)
     - **심각도**(blocker / major / minor)별 목록
     - **만족도 판정**: "사전지식 없이 이 챕터만으로 완주 가능한가?" (satisfied / needs-work)

3. **피드백 기록**: 보고를 `feedback/chNN.md`에 **누적** 기록(라운드 번호·날짜·발견사항·심각도·판정). 기존 내용 위에 새 라운드를 추가.

4. **보완 계획 + 적용**: 보고의 blocker/major부터 `chapters/chNN.md`를 직접 보완.
   - 누락된 설명·사전지식·명령·전제·코드 조각·스크린샷 지시를 추가.
   - **반드시 유지**: 장 템플릿/아이콘 코너 규약(FORMAT_SPEC.md), 그림 `<!-- FIG -->` 마커·번호·순서. 그림 번호를 새로 만들면 안 됨(필요시 기존 그림 지시문 보강).
   - 친근한 대화체, 번역투 금지.

5. **결정적 게이트 + 상태 갱신**: 보완 후 `python tools/grade_chapter.py N` 실행. PASS여야 함(FAIL이면 구조/코드부터 고친다). `feedback/PROGRESS.md`에서 해당 챕터 `round++`, `status` 갱신 — **satisfied는 grader PASS + fresh-LLM satisfied 둘 다일 때만**.

6. 한 이터레이션은 **한 챕터**만 처리(또는 시간이 남으면 2개). 30분마다 다음 챕터로.

## 종료 조건
모든 챕터 `status=satisfied`. 이때 ScheduleWakeup을 더 호출하지 않아 루프가 자연 종료된다.
