# ch18 피드백 — 조이스틱으로 우주선 슈팅

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (개념·흐름은 좋으나 18.4~18.6이 전부 "삽입 위치 미지정 조각"이고 통합 전체 게임 코드가 없어 초보가 완성본을 못 돌림 + 충돌 제거 로직 크래시 가능)

### blocker
1. **통합 전체 게임 코드 부재** — 18.4(총알)·18.5(적·충돌)·18.6(데드존)이 모두 "이벤트 루프 안에 추가", "게임 루프 안" 같은 조각으로만 제시됨. fresh 학습자는 어느 조각을 18.3 뼈대의 어디에 끼워 넣어야 하는지 확정할 수 없어 동작하는 완성본을 못 만듦. → 장 끝에 어디에 무엇이 들어가는지 확정된 완성 `main.py` 전체 코드 제공.
2. **18.6 데드존 이동 코드 ↔ 18.3 이동 코드 관계 불명** — 18.6 블록이 18.3의 `ship_x += int(axis_x * SHIP_SPEED)` 두 줄을 **교체**하는지 추가하는지 명시 없음. 그대로 추가하면 이동이 두 번 적용됨. → "교체"임을 명시하고 통합 코드에 반영.
3. **충돌 판정 이중 제거 → `ValueError` 크래시** — `for bullet in bullets[:]` / `for enemy in enemies[:]` 이중 루프에서 이미 `enemies.remove(enemy)`로 제거된 적을, 같은 프레임의 다른 총알이 또 `enemies.remove(enemy)` 시도 → 리스트에 없으면 `ValueError`. 동일 프레임에 여러 총알이 같은 적 근처면 크래시. → 제거 대상을 집합(`set`)에 모았다가 한 번만 빼는 안전한 로직으로 수정.

### major
4. **충돌 반지름이 작아 총알이 적을 통과** — `dist < (ENEMY_SIZE // 2 + 4)` = `7 + 4 = 11`px. 총알 속도 8px/프레임이라 빠른 적·총알이 한 프레임에 11px를 넘어 스쳐 지나가면 판정을 놓침. → 반지름 합을 넉넉히(적 절반 + 총알 길이 절반 + 여유) 키움.
5. **적-우주선 충돌(게임 오버) 본문 코드 부재** — 18.5/18.6 본문에 게임 오버 판정 코드가 없는데 연습문제·다음 장 예고는 게임 오버를 전제. → 적이 우주선에 닿으면 게임 오버되는 판정·표시 코드를 통합본에 포함.
6. **`import random` 배치 모호** — 18.5 코드 조각 상단에 `import random`이 있어, 파일 맨 위 import 묶음에 둬야 함을 초보가 놓침. → 통합본 상단 import 묶음에 명시 + 본문에 "파일 맨 위로" 안내.

### minor
7. **`int(axis * SHIP_SPEED)` 절삭으로 약한 입력 무시** — `int()`는 0 방향 절삭이라 `axis=0.15`, `SHIP_SPEED=5`면 `int(0.75)=0`이 되어 살짝 기운 입력이 움직임을 못 만듦(데드존 0.1을 통과했는데도). → `round()`로 바꿔 약한 입력도 최소 1px 반영.

### 적용한 보완 (Round 1)
- 18.5 충돌 판정 코드를 **집합 기반 안전 제거**로 교체: `hit_bullets`/`hit_enemies` set에 인덱스를 모았다가 루프 종료 후 한 번만 제거 → 이중 `remove` ValueError 제거. 본문에 "이미 사라진 적을 또 지우려 하면 크래시" 설명 추가.
- 충돌 반지름을 `ENEMY_SIZE // 2 + 6`(넉넉) 수준으로 키워 통과 현상 완화. 본문에 반지름을 키우는 이유 한 줄 추가.
- 18.6 데드존 블록을 18.3 이동 코드의 **교체**임을 명시(본문 + 코드 주석). `int()` → `round()`로 변경(약한 입력 무시 완화), 통합본에도 반영.
- 18.5 `import random`은 "파일 맨 위 import 묶음에 둔다"고 본문에 명시.
- 장 끝(18.7 신설)에 **완성 `main.py` 전체 코드** 추가: 18.2~18.6 조각이 어디에 들어가는지 확정된 단일 실행 파일. 적-우주선 충돌 시 게임 오버(GAME OVER 표시 후 종료) 포함.
- 그림 마커 id·번호·순서(ch18-f01~f16), 아이콘 코너(note/tip/warning/abstract/question/example/info) 전부 유지. 기존 18.1~18.6 절 번호 유지, 통합본은 18.7로 신설(뒤 마무리 섹션 순서 유지).

## Round 2 (2026-06-18) — 하드웨어 연동 실패 시나리오 검증

**판정: needs-work → fixed** (코드 로직은 정상(18.7 동작 확인). 단, 조이스틱이 OS에 안 잡히거나 버튼/축 번호가 다른 컨트롤러일 때 "증상 없이 안 움직이는" 막다른 길이 남아 있었음)

### major
8. **조이스틱 미인식 시 무증상 정지** — 18.3/18.7 코드는 `pygame.joystick.get_count()==0`이면 `joy = None`으로 조용히 넘어감. 그 결과 게임 창은 정상으로 뜨지만 이동 코드(`if joy:`)가 통째로 건너뛰어져 우주선이 꼼짝 안 함. fresh 학습자는 "코드가 다 맞는데 왜 안 움직이지?"에서 막힘(원인이 화면에 아무 단서도 없음). → 시작 시 미인식이면 `print("조이스틱이 안 잡혔습니다 — 17장 게임패드 모드로 꽂았는지 확인하세요.")` 안내 출력 추가 + 본문 트러블슈팅 단락.
9. **버튼 0번/축 0·1번이 컨트롤러마다 다를 수 있음** — 발사 버튼 0번, X=0축·Y=1축을 책이 단정. 키링/펌웨어가 다르면 발사 안 되거나 축이 뒤바뀌는데, 자기 번호를 확인할 경로가 없었음. → 18.2 테스트 코드에 `JOYBUTTONDOWN`에서 `event.button` 출력 추가, "여기서 확인한 내 번호를 게임 코드(`event.button == 0`, `get_axis(0/1)`)에 반영하라"는 연결 안내(⚠️ 코너) 추가.

### 적용한 보완 (Round 2)
- **18.2 테스트 코드**: 이벤트 루프에 `if event.type == pygame.JOYBUTTONDOWN: print(f"버튼 눌림: {event.button}번")` 한 줄 추가 → 발사로 쓸 버튼의 실제 번호를 직접 확인 가능.
- **18.2 본문**: ⚠️ 코너 신설 — 버튼·축 번호가 컨트롤러마다 다를 수 있음을 경고하고, 확인한 번호를 18.3~18.7 코드(`event.button == 0`, `get_axis(0)`/`get_axis(1)`)에 반영하는 방법(번호 교체, 축 스왑) 안내.
- **18.3 / 18.7 코드**: 조이스틱 초기화에 `else:` 분기 추가 — 미인식 시 안내 메시지 출력. 인식 시에는 `get_name()` 출력으로 어떤 컨트롤러가 잡혔는지 확인.
- **18.3 본문**: ⚠️ "창은 떴는데 우주선이 안 움직인다면" 트러블슈팅 단락 추가 — 17장 게임패드(HID) 모드 확인, `joy.cpl` 장치 확인, 18.2 재실행으로 `get_count()` 점검.
- 그림 마커·번호·순서, 코너 아이콘, 절 번호 전부 유지(추가는 본문 코너/코드 라인뿐, 신규 그림 없음). 세 개 전체 코드 블록(18.2/18.3/18.7) `ast.parse` 통과 재확인.
