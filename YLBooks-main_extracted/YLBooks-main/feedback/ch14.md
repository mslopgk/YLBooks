## Round 3 (2026-06-18) — 차트 에디터(14.5) 실동작 결함 수정

### 판정: 개선 적용 (본문이 약속한 되감기·자동재생이 실제로 동작하게 만듦)

### blocker
1. **`get_pos()`로 곡 위치를 재서 되감기·자동재생이 깨짐** — `pygame.mixer.music.get_pos()`는 "맨 처음 `play()` 이후 누적 경과(ms)"라 `play()`로 되감아도 0으로 리셋되지 않고 계속 증가. 그래서 (a) 스페이스바로 되감아도 `now`가 안 줄어 노트가 엉뚱한 위치로 흐르고, (b) 곡 끝 자동재생 후 `now`가 곡 길이를 넘어 계속 커져 모든 노트가 화면 밖으로 사라짐. 본문이 약속한 두 동작이 둘 다 거짓이 됨. → **별도 타이머로 교체**: `play()`/되감기/자동재생 때마다 `play_start = pygame.time.get_ticks()`를 다시 찍고 `now = (get_ticks() - play_start)/1000`으로 곡 위치 계산. `get_busy()`로 곡 끝 감지 시 `play()`+타이머 리셋. (ch11 게임도 동일하게 `get_ticks()` 기반 타이밍을 씀 — 일관.)
2. **좌클릭으로 찍은 노트가 다음 프레임에 화면 밖으로 사라져 안 보임** — 찍은 노트는 곧 판정선을 지나 화면 밖으로 흘러감(낙하식 타임라인의 정상 동작이지만, 방금 찍은 것을 못 봐서 편집이 불편). → 화면 오른쪽에 **"찍은 노트" 목록 패널을 항상 표시**(시간순 최근 12개, `MM.MMs L레인`). 본문·팁에도 이 패널 설명 추가.

### major
3. **좌표 변환이 본문 설명과 반대 방향** — 기존 `time_to_y`는 `+SECONDS_ON_SCREEN` 때문에 `now`가 화면 **위쪽**에 오고 아래 판정선이 "4초 전 과거"였음. 그 결과 본문이 말한 "아래 판정선=지금, 노트가 위에서 판정선으로 흘러내림"과 어긋났고, 판정선 클릭이 항상 과거(`now-4s`) 시각에 노트를 찍어 즉시 화면 밖으로 나감(=blocker 2의 근본 원인). → `time_to_y = HEIGHT - (note_time-now)/S*HEIGHT`, `y_to_time = now + (HEIGHT-y)/S` 로 보정. 이제 `now`가 아래 판정선(y=HEIGHT), 미래 노트가 위에서 내려오고, 판정선 클릭이 곧 `now`에 노트를 찍음 — 코드가 본문 설명과 일치.

### minor
4. 팁 박스를 "`get_busy()`로 끝 감지" 한 줄에서 → "`get_pos()` 누적 함정 + `play_start` 타이머 리셋" 정확한 설명으로 교체. 패널 변수명 `i` 중복 사용을 `row`로 변경(레인 구분선 루프와 셰도잉 회피).

### 검증 (헤드리스)
- 책에 실린 `chart_editor.py` 블록을 그대로 추출 → `py_compile` 통과(122줄).
- 더미 SDL 드라이버 + mixer 스텁으로 실제 루프 구동: 재생 중 좌클릭이 **양수 시각**(0.13s·0.26s)에 노트 추가, 우클릭 삭제 동작, 곡 끝 감지 시 자동재생으로 타이머 0초 리셋, 스페이스 되감기로 타이머 즉시 0초 리셋, `play()` 호출 횟수 3회(초기+자동+되감기) 확인 → **HEADLESS RUNTIME OK**. (대조: 옛 `get_pos()`였다면 같은 시나리오에서 `now`가 ~7.96s로 계속 증가, 리셋 안 됨.)
