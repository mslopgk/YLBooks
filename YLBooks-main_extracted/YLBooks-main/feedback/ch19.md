# ch19 피드백 — 적과 폭발 효과

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (개념 흐름·코너·그림은 양호하나, 핵심 객체 모델이 정의 없이 사용되어 초보가 코드를 돌릴 수 없음)

### 핵심 배경
- 18장은 총알·적을 **`[x, y]` 리스트**로 관리하고, 충돌은 **중심점 거리**로 판정한다. 18장에는 `Player`/`Enemy`/`Bullet`/`Particle` **클래스가 전혀 없고**, `.rect`·`.alive`·`player.bullets`·`spawn_enemies`도 없다.
- 그런데 19장은 첫 절(19.1)부터 `bullet.rect`/`enemy.rect`/`bullet.alive`를, 19.6 통합 코드에서 `player.bullets`·`spawn_enemies`를 정의 없이 사용한다. → 19장이 "리스트 → 클래스" 전환을 명시·제공하지 않으면 fresh 학습자는 AttributeError/NameError로 즉시 막힌다.

### blocker
1. **`spawn_enemies` 정의 부재** — 19.6 `main.py`가 `from enemy import Enemy, spawn_enemies`로 import하고 `spawn_enemies(enemies, pygame.time.get_ticks())`로 호출하지만, 함수 정의가 챕터 어디에도 없음. 적 생성은 이 게임의 핵심 → 19.5에 `spawn_enemies` 정의(난이도 곡선의 `enemy_speed`를 사용해 적을 spawn) 추가.

### major
2. **`.rect` 순서 역전 + 클래스 모델 부재** — 19.1에서 `bullet.rect`/`enemy.rect`/`bullet.alive`를 쓰지만 `Enemy.rect` 정의는 19.5에야 나오고, `Bullet`/`Player`(및 `player.bullets`) 클래스는 챕터 어디에도 정의가 없음. → 19.1 도입부에 "18장의 `[x,y]` 리스트를 클래스로 바꾼다"는 전환 설명 + `Bullet`·`Player`(`.rect`/`.alive`/`bullets` 포함) 정의 코드 블록을 충돌 코드 **앞**에 배치.
3. **Particle 코드 블록 `import pygame` 누락** — 19.2 블록은 `import random`/`import math`만 있는데 `draw`에서 `pygame.draw.circle`을 사용 → 그 블록만 실행하면 NameError. → `import pygame` 추가.
4. **폰트 크래시 무방비** — 19.4 `pygame.font.Font("NanumGothic.ttf", 28)`는 파일이 없으면 즉시 예외로 게임이 안 켜짐(사운드는 try/except 안내가 있는데 폰트는 없음). → 폰트 로드를 try/except로 감싸 시스템 기본 폰트로 폴백하는 코드·설명 추가.
5. **최종 19.6 `main.py` 통합 누락** — 사운드 재생(shoot/explode), HUD(점수·목숨), 난이도(`enemy_speed`)가 모두 빠져 있어 "완성형"이라는 본문 주장과 불일치. 또 폭발 시 `explode_sound.play()`도 없음. → 통합 `main.py`에 사운드 로드/재생, HUD 그리기, 난이도 곡선, 발사음, 목숨/게임오버 흐름 반영.

### minor
6. **죽은 `dt` 변수** — 19.6 `dt = clock.tick(60)`로 받지만 어디에도 사용 안 함. → `clock.tick(60)`로 단순화(반환값 미사용).
7. **score 중복 정의** — 19.4에서 `score = 0`, 19.6 초기화에서도 `score = 0`. 통합 코드가 단일 출처가 되도록 19.4 블록은 "HUD 그리기" 위주로 남기고 초기화는 19.6에 집중.
8. **총알-다중적 충돌 `break` 부재** — 19.1·19.6 이중 반복문에서 총알 하나가 여러 적과 동시에 겹치면 한 총알이 여러 적을 죽임(의도 모호). 18장은 `break`로 "총알 하나 = 적 하나"를 보장했음. → 안쪽 루프에 `break` 추가로 18장과 일관.

### 적용한 보완 (Round 1)
- **19.1**: 도입부에 18장 `[x,y]` 리스트 → 클래스 전환 설명 추가. 충돌 코드 앞에 `Bullet` 클래스(`rect`/`alive`/`update`/`draw`)와 `Player` 클래스(`bullets` 리스트 보유, 발사 메서드)를 정의하는 코드 블록 신설. 이중 반복문 충돌에 `break` 추가.
- **19.2**: Particle 코드 블록 상단에 `import pygame` 추가.
- **19.4**: 폰트 로드를 `try/except`로 감싸 `pygame.font.SysFont` 폴백 제공(파일 없어도 게임이 켜짐). 중복 `score = 0`은 HUD 블록에서 제거하고 19.6 초기화로 일원화.
- **19.5**: `spawn_enemies(enemies, now_ms)` 함수 정의 추가 — 내부 spawn 타이머(밀리초 기반)로 일정 간격마다 난이도 곡선 `enemy_speed`를 계산해 `Enemy(speed)`를 append. `Enemy.__init__`이 `x`도 인자로 받도록(랜덤 X) 보강.
- **19.6**: 통합 `main.py`를 사운드 로드/재생(`shoot_sound`/`explode_sound`, try/except), HUD(점수·목숨, 폰트 폴백), 난이도(`spawn_enemies`가 시간 인자로 속도 조절), 폭발음 `explode_sound.play()`, 목숨 감소·간단 게임오버 흐름까지 포함하도록 재작성. 죽은 `dt` 제거(`clock.tick(60)`).
- 그림 마커 id·번호·순서(ch19-f01~f16) 전부 유지. 아이콘 코너(note/tip/warning/abstract/question/example/info) 전부 유지. 절 번호 변경 없음.

## Round 2 (2026-06-18) — 실제 코드 버그 수정

**판정: 실행 검증 완료** (3개 모듈 + main.py를 pygame 2.6.1 헤드리스로 실제 실행해 동작 확인)

### blocker
9. **목숨이 절대 안 깎이는 데드 로직** — `Enemy.update()`가 화면 밖 적을 즉시 `alive=False`로 만드는데, 메인 루프 목숨 차감 조건이 `enemy.alive and enemy.rect.top > SCREEN_HEIGHT`라 항상 False. → 목숨이 영영 0이 되지 않아 게임오버 도달 불가. (격리 테스트로 옛 조건이 탈출한 적 3개 중 0개만 매치함을 확인 → 버그 재현.)

### major
10. **폰트 예외 누락 케이스** — `except FileNotFoundError`만으로는 pygame이 폰트 로드 실패 시 던지는 `OSError`를 못 잡아 게임이 크래시할 수 있음.
11. **난이도/생성 상수 정의 위치 불명** — 19.6 `main.py`가 `BASE_SPEED`/`SPEED_RATIO`/`SPEED_UP_SEC`/`SPAWN_INTERVAL_MS`를 쓰지만 어느 모듈에 정의되는지 본문에 명시 안 됨 → import 시 `NameError` 위험. `Bullet` 클래스의 소속 파일도 불명확.

### 적용한 보완 (Round 2)
- **19.5 Enemy**: `escaped` 플래그 추가. `update()`에서 `rect.top > SCREEN_HEIGHT`면 `escaped=True` + `alive=False`. "탈출 사실은 alive와 분리해 기억한다"는 설명 단락 추가.
- **19.5 상수**: `BASE_SPEED`/`SPEED_RATIO`/`SPEED_UP_SEC`/`SPAWN_INTERVAL_MS`를 `enemy.py` 모듈 최상단(클래스 위)에 실제로 정의. `spawn_enemies` 블록의 중복 `SPAWN_INTERVAL_MS=800`은 제거하고 "같은 파일이라 상수 공유" 주석으로 대체.
- **19.6 목숨 로직**: 차감 조건을 `if enemy.escaped: lives -= 1; enemy.escaped = False`로 교체(alive 여부와 분리). 코드 아래에 버그 원인·수정 설명 단락 추가. → 헤드리스 실행에서 탈출 적 3개 → lives 3→0 → "GAME OVER reached" 확인.
- **폰트 폴백**: 19.4·19.6 두 곳 모두 `except (FileNotFoundError, OSError)`로 확장 + SysFont 폴백 유지. 19.4 본문에 "OSError도 함께 잡는 이유" 설명 추가.
- **파일 배치 명시**: 19.6 import 줄에 주석(`Bullet`은 `player.py`, 난이도 상수는 `enemy.py`), 본문에 `Bullet`/`Player`→player.py, `Enemy`/`spawn_enemies`/상수→enemy.py, `Particle`/`spawn_explosion`→particle.py 명시. 리팩터 프롬프트 예시도 `Bullet 클래스`·`spawn_enemies와 난이도 상수`를 포함하도록 갱신.
- **죽은 `dt`**: 현재 chapters/ch19.md에는 이미 `clock.tick(60)`만 있고 `dt`는 존재하지 않음(Round 1에서 제거됨) → 추가 조치 불필요.
