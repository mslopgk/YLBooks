## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (UI/등급 개념 설명은 좋으나, 폰트 파일명 대소문자 불일치로 맥/리눅스 크래시, 맥·리눅스 폰트 확보법 부재, 15.5 시작 화면이 산문뿐 동작 코드 없음)

### blocker
1. **폰트 파일명 대소문자 불일치** — 코드/경로는 `Malgun.ttf`(대문자, 15.2 `font.Font`·`FONT_PATH`·그림 15.6)인데, 본문 "안전한 방법"에서는 `C:\Windows\Fonts\`의 `malgun.ttf`(소문자)를 복사하라고 함. 윈도우는 대소문자 무시라 우연히 동작하지만, 맥/리눅스는 대소문자를 구분해 `pygame.font.Font` 로드 실패로 크래시. 게다가 ch19는 `NanumGothic.ttf`를 "15장에서 설치한 것"으로 전제 → 책 전체 일관성이 깨짐. → ch15을 **나눔고딕(`assets/fonts/NanumGothic.ttf`)으로 통일**(코드·캡션·요약 전부). 한 글꼴 이름으로 일원화하고 `FONT_PATH` 한 곳에서 관리.
2. **맥/리눅스용 폰트 확보 방법 부재** — 분기 코드의 리눅스 가지는 `font_path = None`(따로 설치, 끝)뿐이고, 재배포 가능 폰트(나눔고딕)를 *어디서 어떻게* 받는지 다운로드처가 전혀 없음 → 윈도우 외 학습자는 막힘. → 공식 배포처 2곳 안내(네이버 한글한글아름답게 `hangeul.naver.com`, 구글 폰트 `fonts.google.com/specimen/Nanum+Gothic`)와 리눅스 `sudo apt install fonts-nanum` 추가. URL은 지어내지 않고 공식 도메인/페이지 경로만 명시. 라이선스(OFL, malgun과 달리 재배포 가능) 이유도 한 줄 설명.

### major
3. **15.5 "시작 화면"이 산문뿐, 동작 코드 0줄** — 시작 화면 그리기·곡/난이도 선택 키·`restart_game()`·결과 화면 호출·곡 종료 자동 전환이 전부 글로만 설명되고 코드가 없음. 15.4 끝의 "게임 루프 끝에서 호출하고 R로 재시작 연결" 도 말뿐. → 네 화면을 `state`("start"|"play"|"result") + `paused` 로 묶은 **그대로 돌아가는 뼈대 코드**를 15.5에 추가: `draw_start_screen`, `reset_play_state`/`restart_game`, 이벤트 분기(↑↓ 곡, 1~5 난이도, SPACE 시작, P 일시정지, R 재시작, Q 종료), `song_finished()` → `state="result"` 자동 전환, 일시정지 오버레이. `update_game/draw_game/song_finished`는 14장 로직을 넣는 자리로 명시.

### minor
4. **`FONT_PATH`/`font_path` 변수명 혼선** — 15.2 분기 코드는 `font_path`(소문자), 최종 권장 코드는 `FONT_PATH`(상수) → 둘이 같은 건지 초보가 헷갈림. → 최종 권장 코드에서 "이제부터 `FONT_PATH` 한 이름으로 관리" 라고 명시해 정리.
5. **`import sys` 누락** — 15.5 옛 코드가 `sys.exit()`를 쓰는데 `import sys` 없음. → 통합 뼈대 상단에 `import sys`, `import pygame` 추가.
6. **등급색 2분기** — 15.4 `grade_color = COLOR_PERFECT if grade=="S" else COLOR_COMBO` 라 A/B/C가 전부 같은 노란색. → `GRADE_COLORS` 딕셔너리로 S 초록 / A 파랑 / B 노랑 / C 빨강 4색 분기.

### 적용한 보완 (Round 1)
- 15.2: 폰트를 **나눔고딕으로 통일**(`SysFont→Font` 예시, 그림 15.3/15.6 캡션, `FONT_PATH`). OS 분기 코드의 리눅스 가지를 `None` 대신 나눔고딕 경로로 바꾸고 `sudo apt install fonts-nanum` 안내 추가. "안전한 방법"에 나눔고딕 공식 배포처 2곳(네이버 한글한글아름답게, 구글 폰트)·OFL 라이선스·malgun 재배포 불가 이유 보강. `FONT_PATH` 일원화 설명 추가.
- 15.4: 등급색을 `GRADE_COLORS`(S/A/B/C 4색) 딕셔너리로 분기. 결과 화면 호출 위치는 15.5에서 통합한다고 연결.
- 15.5: 네 화면을 `state`+`paused`로 묶은 **실행 가능한 통합 뼈대 코드** 추가 — `import sys/pygame`, `draw_start_screen`, `reset_play_state`/`restart_game`, 이벤트 분기(곡·난이도·SPACE·P·R·Q), `song_finished()`→결과 화면 자동 전환, 일시정지 오버레이. 어디에 14장 로직(`update_game/draw_game/song_finished`)을 넣는지 명시. 동작 요약 4항목 추가. Antigravity 요청은 "직접 옮기는 대신" 선택지로 재배치.
- 15.1: 비포·애프터 캡션의 "말굿 폰트"→"나눔고딕 폰트"로 일관화.
- 그림 마커·번호·순서(f01~f16) 전부 유지. 아이콘 코너(note/tip/warning/abstract/question/example/info) 유지. PROGRESS.md 미변경.

## Round 2 (2026-06-18) — fresh 학습자 + 결정적 grader

**판정: FAIL (blocker)** — Round 1에서 추가한 코드가 미정의 이름을 그대로 사용해 실행 시 NameError. grader가 정적 검사로 잡음.

### blocker
1. **미정의 이름 4종** — `SCREEN_W`/`SCREEN_H`(화면 크기), `song_finished()`(곡 종료 검사), `last_judge`(최근 판정)가 챕터 어디에도 정의되지 않은 채 15.3·15.4·15.5 코드에서 사용됨 → 그대로 실행 시 `NameError`. 15.5 뼈대도 이 이름들에 의존해 "그대로 돌아가는 뼈대"가 실제로는 안 돎.

### major
2. **폰트 변수명 3중 분열** — 15.2는 `font_large/medium/small`, 15.3은 `font_score/combo/ui`, 결과·뼈대 코드는 `font_large/medium/ui` 사용. 셋이 달라 어떤 이름이 진짜인지 학습자가 혼란 + 뼈대 실행 시 미정의.

### minor
3. 폰트 초기화가 `pygame.init()`보다 먼저 오면 오류 — 순서 안내 부재. `import sys` 존재 여부 재확인 필요.

### 적용한 보완 (Round 2)
- 15.2: 폰트 객체 이름을 **`font_large`(48)/`font_medium`(32)/`font_ui`(16)** 한 세트로 확정(기존 `font_small`→`font_ui`). "이 장 모든 코드가 이 세 이름을 쓴다" 명시. `pygame.font.Font`는 `pygame.init()` 다음에 호출하라는 ⚠️ 주의 박스 추가.
- 15.3: 새 폰트 변수(`font_score/combo/ui`) 정의를 삭제하고 15.2의 세 이름을 재사용하도록 설명만 남김. 판정 표시 스니펫에 `SCREEN_W, SCREEN_H = 480, 800` 상수와 `last_judge = ""` 초기화를 명시하고, `if last_judge:` 가드로 첫 프레임 안전 처리.
- 15.5 뼈대: 상단에 `SCREEN_W/SCREEN_H` 상수, 팔레트 6색, `FONT_PATH`, 게임 상태(`score/combo/max_combo/perfect/good/miss/last_judge/song_started`) 선언을 모두 포함. **`song_finished()` 기본 구현**(곡 시작 후 `SONG_LENGTH`=60초 경과 검사) 제공. `reset_play_state()`가 `song_started=get_ticks()`로 시작 시각 기록. `pygame.init()` **다음에** `font_large/medium/ui` 생성(순서 주석). `import sys`/`import pygame` 확인. 결과 화면 함수는 15.4에서 정의해 이 위치에 둔다고 주석으로 연결.

## Round 3 (2026-06-18) — "그대로 실행" 단언 검증

**판정: needs-work → 수정완료** — 15.5 뼈대가 결과 화면 함수 3종을 호출하지만 블록에 미포함(복붙 시 result 상태 NameError) + 폰트 파일 없으면 즉사. "그대로 실행됩니다" 단언이 거짓이던 두 결함을 수정.

### blocker
1. **결과 화면 함수 3종 미포함** — 15.5 뼈대 루프가 `state=="result"`에서 `draw_result_screen(...)`을 호출하고 그 안에서 `calculate_accuracy`/`calculate_grade`를 부르는데, 셋 다 15.4에서만 정의되고 뼈대 코드 블록에는 없었음. 자리에 주석("15.4에서 이미 정의했습니다…")만 있어, 뼈대 블록만 복붙하면 곡 종료 직후 result 상태에서 즉시 `NameError`. → "그대로 실행됩니다" 단언과 모순.

### major
2. **폰트 파일 없으면 즉사** — `pygame.font.Font(FONT_PATH, …)`를 try 없이 호출. `assets/fonts/NanumGothic.ttf`가 아직 없으면(독자가 폰트 다운로드 전에 "그대로" 실행) 첫 폰트 로드에서 `FileNotFoundError`(맥/리눅스 손상 파일 등은 `OSError`)로 창도 못 뜨고 죽음.

### minor
3. **`last_judge` 선언 중복** — 모듈 최상단(3번 블록)과 `reset_play_state()`가 둘 다 `last_judge=""`를 설정해 어느 쪽이 "진짜 선언"인지 모호.

### 적용한 보완 (Round 3)
- 15.5 뼈대: `calculate_accuracy`/`calculate_grade`/`draw_result_screen` 세 함수를 **뼈대 블록 안('6) 15.4 함수 합치기')에 그대로 인라인** — 블록 단독 복붙으로도 result 상태가 NameError 없이 돈다. 폰트 정의 자리('5)' 주석)에는 "이 세 함수를 여기 두면 된다 + 없으면 NameError"를 명확히 안내.
- 15.5 뼈대: 폰트 로드를 `load_font(size)` 헬퍼로 감싸 `try/except (FileNotFoundError, OSError)` → 실패 시 `pygame.font.SysFont(None, size)` 폴백 + 경고 메시지("assets/fonts/에 NanumGothic.ttf를 넣으세요"). 두 예외 모두 포착.
- 15.5 뼈대: `last_judge` 선언을 최상단 한 곳으로 정리(주석 "선언은 여기 한 곳뿐"). 최상단 3번 블록은 "이름만 만들어 둔다, 실제 초기화는 reset_play_state()가 도맡는다"로 의도 명시.
- 15.5 본문: "그대로 돌아가는 뼈대" 문장에 "15.4의 세 함수까지 함께 실어 두었으니 이 블록 하나만 복사해 실행해도 시작→결과 흐름이 끝까지 돈다"를 추가해 단언과 코드 일치.
- 검증: python 블록 8개 전부 `ast.parse` 통과. 가짜 pygame(폰트 파일 없음 시뮬레이션)으로 뼈대 실행 → 폰트 폴백 3회 발동 + DOWN/2/SPACE→play→(60초 경과)→result→Q 종료까지 **NameError 0건** 완주 확인.
- 그림 마커·번호·순서(f01~f16) 유지. 아이콘 코너 유지. PROGRESS.md 미변경.
