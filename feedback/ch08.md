# ch08 피드백 — 4버튼 키링 완성 (리듬게임용 입력 장치)

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (스케치 저장 규칙·라이브러리 설치 안내 누락으로 장 단독 완결 불가)

### blocker
1. **스케치 저장 규칙 누락** — 209행 "코드를 스케치 폴더에 저장하고"만 있고, 폴더명=`.ino` 파일명 규칙·확장자 `.ino`·저장 위치(`keyring4` 폴더)가 명시되지 않음. 5장(155행)에서는 "폴더 이름과 `.ino` 파일 이름은 같아야 한다"를 명시했으나 8장에는 그 안내가 없어 처음 4버튼 코드를 저장하는 fresh 학습자가 `keyring4/keyring4.ino` 구조를 만들지 못함. → 완성 코드 뒤에 저장 규칙(폴더 `keyring4`, 그 안에 `keyring4.ino`) 명시.
2. **Bounce2 라이브러리 설치 안내 없음** — 완성 코드는 `#include <Bounce2.h>`를 쓰는데 8장에는 설치 명령도, 7장 참조도 없음. 7장을 건너뛰고 8장만 펼친 독자는 컴파일 시 `Bounce2.h: No such file or directory` 에러를 만남. → `arduino-cli lib install "Bounce2"` 명령 + 7장 참조 + 미설치 시 에러 안내 추가.

### major
3. **보드 표기 불일치 (Leonardo vs Pro Micro)** — 프롬프트(117행)·완성 코드 주석(158행)은 "Arduino Leonardo (ATmega32U4)"라고 쓰는데, 실제 compile/upload FQBN(212~213행)은 `arduino:avr:micro`(Arduino Micro / Pro Micro 호환)임. 5장에서 이 책 기본 보드를 `arduino:avr:micro`로 잡았으므로 8장도 같은 32U4 계열임을 한 줄 설명하거나 표기를 통일해야 혼동이 없음. → 둘 다 같은 ATmega32U4 계열임을 한 줄 설명 + 코드 주석을 5장 기본값에 맞춰 정리.
4. **D10 표 vs 코드 모순** — 표 8.1(52행)은 "D10 = 예약 / 안전장치용으로 비워 둠"인데, 프롬프트·완성 코드(124, 173~179행)는 D10을 OUTPUT + LOW로 적극 사용(GND 핀). "비워 둠"과 "적극 사용"이 정면 충돌. → 표 8.1 D10 행을 코드 용도(OUTPUT LOW로 보조 GND 고정)와 정합하도록 수정.

### minor
5. **COM 포트 확인 방법 부재** — 209행은 "`COM3`는 자기 환경의 번호로 바꾼다"만 있고 어떻게 확인하는지 없음. 5장 `arduino-cli board list` 한 줄 참조 보강.
6. **디바운스 인터벌 수정 위치 미지정** — 282행·연습문제 3번에서 "10ms→15~20ms로 늘려라"라고 하면서 코드 어디를 고치는지 안 짚음. → `bouncers[i].interval(10);`(완성 코드 185행) 한 줄임을 명시.
7. **한/영 입력 상태 함정** — 메모장 테스트(223행) 시 IME가 한글이면 `d f j k`가 한글 자모로 입력되거나 조합돼 "안 된다"고 오판 가능. → 영문 입력 상태에서 확인하라는 주의 한 줄 추가.

### 적용한 보완 (Round 1)
- 8.4절에 **Bounce2 설치 안내**(7장에서 설치, `arduino-cli lib install "Bounce2"`, 미설치 시 `Bounce2.h: No such file or directory` 에러) 박스 추가 [blocker 2].
- 완성 코드 뒤 본문에 **스케치 저장 규칙**(폴더 `keyring4` 안에 같은 이름 `keyring4.ino`, 5장 규칙 재확인) 명시 [blocker 1].
- 프롬프트·완성 코드 주석의 보드 표기를 정리하고 **"Leonardo·Micro·Pro Micro 모두 ATmega32U4 같은 계열 → FQBN `arduino:avr:micro`로 통일"** 한 줄 설명 추가 [major 3].
- 표 8.1 **D10 행**을 "예약/비워 둠" → "보조 GND로 OUTPUT LOW 고정(코드에서 사용)"으로 수정해 코드와 정합화 [major 4].
- compile/upload 직전 **COM 포트 확인**(`arduino-cli board list`, 5.4절 참조) 한 줄 보강 [minor 5].
- 8.6 고스트 입력 문단·`⚠️ 주의` 박스에 **디바운스 수정 위치**(완성 코드 `bouncers[i].interval(10);`) 명시 [minor 6].
- 메모장 테스트 문단에 **영문 입력 상태(IME 한/영) 주의** 한 줄 추가 [minor 7].

## Round 2 (2026-06-18) — fresh 학습자 검증

### major
1. **D10/D16 핀 기능 서술 부정확 (45행)** — "D10/D16은 SPI·I²C"는 ATmega32U4에서 틀림. D10은 SPI/I²C 핀이 아니며(표 8.1에서 D10=보조 GND), 실제로 SPI는 D14~D16(MISO/SCK/MOSI), I²C는 D2/D3(SDA/SCL)임. → 본문 핀 기능 서술을 32U4 실제 핀맵에 맞게 정정하고, D2~D5를 버튼 입력으로만 쓰므로 충돌 없음을 명확히. 표 8.1의 "D16은 SPI SCK" 주석도 부정확(D15=SCK, D16=MOSI) → "SPI(MISO/SCK/MOSI)로도 쓰임"으로 정정.
2. **폴더/`.ino` 생성 도구·arduino-cli 전제 미안내** — 저장 규칙은 있으나 어떤 도구로 폴더·`.ino`를 만드는지(에디터/메모장) 한 줄, arduino-cli가 5장에서 설치됐다는 전제 재안내가 없어 이 장만 본 독자가 막힐 수 있음. → 메모장/텍스트 편집기로 새 파일 생성·`.ino` 저장(메모장은 "모든 파일") 한 줄 + "컴파일·업로드는 5장에서 설치한 arduino-cli로(미설치 시 5장 참조)" 한 줄 추가.

### minor
3. **`Keyboard.h` 설치 불필요 미언급** — Bounce2 설치 안내는 있으나 `Keyboard.h`가 avr 코어 내장이라 별도 설치가 필요 없다는 안내가 없어 독자가 추가 설치를 걱정할 수 있음. → Bounce2 주의 박스에 한 줄 추가.
4. **체크리스트 ③④ "4번" 표현이 for 루프(1회 작성)와 어긋남** — 본문 설명이 "4번 실행/모든 인덱스"라 한 번만 적는 for 루프와 인상이 충돌. → "for 루프면 한 줄만 적어도 4번 돈다, 확인할 것은 반복 범위 `i < 4`"로 표현 보정(제목도 "4번"→"4핀 모두").
5. **COM 포트 확인법** — 이미 `arduino-cli board list`(5.4절 참조) 안내가 본문에 있어 별도 보강 불필요(Round 1에서 반영됨). 인접 문장에 arduino-cli 전제 한 줄을 더해 흐름 자연화.

### 적용한 보완 (Round 2)
- 8.2절 셋째 이유 문단의 핀 기능 서술을 32U4 실제 핀맵(UART D0/D1, SPI D14~16, I²C D2/D3, PWM D6)으로 정정, D2~D5는 버튼 전용이라 충돌 없음 명시 [major 1].
- 표 8.1 D14~D16 주석을 "D16은 SPI SCK" → "SPI(MISO/SCK/MOSI)로도 쓰임"으로 정정 [major 1].
- 스케치 저장 문단에 메모장/텍스트 편집기로 `.ino` 만드는 법(메모장 "모든 파일") + arduino-cli 5장 설치 전제 한 줄 추가 [major 2].
- Bounce2 주의 박스에 `Keyboard.h`는 avr 코어 내장이라 설치 불필요 한 줄 추가 [minor 3].
- 체크리스트 ③④ 본문 설명을 for 루프 1회 작성·반복 범위 `i < 4` 확인 중심으로 보정 [minor 4].
- 유지: 아이콘 코너, 그림 `<!-- FIG -->` 마커·번호·순서, 친근한 대화체 그대로.
