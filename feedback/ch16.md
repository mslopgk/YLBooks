# ch16 피드백 — 조이스틱 핀맵과 데드존 처리

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (개념·실습 흐름은 탄탄하나 "D7=7" 표기 불일치가 같은 장의 "A0≠0" 경고와 충돌해 혼란 유발, 경계 수치 1 오차)

### major
1. **"SW → D7"이라 부르며 코드는 `#define JOY_SW 7` → 표기 불일치.** 같은 16.2에서 "A0과 0은 다른 핀"이라 경고해 둔 탓에 학습자가 "그럼 D7도 7이 아닌 다른 핀 아닌가?"라고 혼란. → 16.2 ⚠️ 주의 박스에 "D7 = 디지털 핀 7번, 디지털 핀은 `A` 접두어 없이 숫자만 써도 되므로 코드에서는 `7`로 적는다(D7=7은 같은 핀). 아날로그 핀만 `A0`처럼 `A`를 붙여 구분" 한 단락 추가로 통일·해소.

### minor
2. **16.5 경계 수치 543 → 542 오차.** 데드존이 `abs(value-center) < deadzone`(`<`)이므로 중앙 512 + 30 = 542가 데드존 밖으로 통과되는 첫 값. "543 이상"은 1 어긋남. → "542 이상"으로 수정하고 512+30=542 산출 근거를 괄호로 명시.
3. **데드존 `<` vs 그림 16.7 "±30" 경계 모호.** 부등호가 `<`라 거리가 정확히 30인 482·542는 데드존 밖인데, 본문/그림의 "±30"만 보면 경계 포함 여부 불명확. → 16.4 설명에 고정 구간(483~541)과 경계값(482·542는 통과) 명시, "±30은 중앙 고정 구간의 폭"이라고 보강.
4. **스케치 폴더/.ino 생성이 5장 의존(자기완결성 부족).** "5장에서 했던 것처럼"만 있고 fresh 학습자는 폴더명=ino명 규칙을 모름. → 16.3에 "폴더 이름과 메인 `.ino` 파일 이름이 같아야 arduino-cli가 스케치로 인식(예: `Joystick/Joystick.ino`)" 한 줄 추가.
5. **`-c baudrate` 형식 설명 부재.** `arduino-cli monitor ... -c baudrate=9600`의 `-c`(이름=값 config)와 `Serial.begin()` 숫자 일치 필요성을 설명 안 함. → 16.3에 `-c`=설정 옵션, baudrate는 `Serial.begin`과 동일해야 글자 안 깨짐, 변경 시 양쪽 동시 수정 안내 추가.
