# ch12 피드백 — 키링으로 노트 치기(판정 시스템)

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (note 객체 정의 부재 + 완성 게임 루프 부재 + 함수 시그니처/상태 변수 불일치로 fresh 학습자가 코드를 합쳐 실행할 수 없음)

### blocker
1. **`note`/`notes` 객체 정의 전무** — 본문 곳곳이 `note.lane`, `note.hit`, `note.time`을 쓰는데 note가 클래스인지 딕셔너리인지, 이 속성들이 어디서 어떻게 채워지는지 한 번도 설명하지 않음. fresh 학습자는 `judge()`도 자동 MISS 루프도 실행 불가. → 12.2 앞에 이 장에서 쓰는 note 구조(`Note` 클래스: `.lane`, `.time`, `.hit`)를 명시하고, `notes` 준비(11장에서 만든 노트 데이터를 `Note` 객체 리스트로) 코드 추가.
2. **조각을 합친 완성 게임 루프 부재** — 입력 처리, judge, handle_result, 자동 MISS, 그리기 타이머가 각각 흩어진 스니펫으로만 존재. 이걸 어떻게 하나의 루프로 합치는지 통합 코드가 없어 "동작하는 게임"에 도달 불가. → 12.6 뒤(또는 12.7 직전)에 상태 초기화 + 전체 게임 루프 통합 코드 추가.

### major
3. **`judge()` 시그니처 불일치** — line77 호출은 `judge(lane, press_time)`(2인자), line155 정의는 `judge(lane, press_time, notes)`(3인자). 호출이 깨짐. → 호출부를 `judge(lane, press_time, notes)`로 통일.
4. **`handle_result()` 불일치** — line78 호출은 `handle_result(result, lane)`(반환 안 받음), line216 정의는 `handle_result(result, score, combo)` → `(score, combo)` 반환. 인자·반환 모두 어긋남. → 정의를 `(result, score, combo)`로 통일하고 호출부에서 `score, combo = handle_result(...)`로 반환을 받도록 정리. lane 강조/판정글자 타이머는 호출부에서 처리.
5. **상태 변수 초기화 전무** — `running/score/combo/judge_text/judge_text_until/lane_lit_until/combo_bump_until` 등이 본문 어디서도 초기화되지 않음. → 통합 코드 도입부에 상태 변수 초기화 블록 추가.

### major (경계 모순)
6. **MISS 경계 모순** — 표(line109) "±100ms 초과 MISS" vs 자동 MISS 코드 `MISS_MS=200`(line275) vs 그림 12.7 "±200ms". 같은 MISS인데 100/200이 충돌. → **키입력 MISS 경계(GOOD_MS=100ms 초과)** 와 **자동 MISS(노트가 판정선을 MISS_MS=200ms 지나칠 때까지 입력 없음)** 를 서로 다른 개념으로 명확히 구분. 12.3 본문/표·그림 12.7 캡션·12.6 본문에 "키입력은 ±100ms, 자동 처리는 통과 후 200ms"를 일관되게 기술.

### minor
7. line274 `elapsed = current_time` 무의미한 재대입 — 제거하고 `current_time - note.time`을 직접 사용.
8. line275 `and not note.hit` 중복 — 바로 위 `if note.hit: continue`로 이미 걸렀으므로 제거.
9. 콤보/레인 강조 "그리기" 적용 코드 누락 — `combo_font_size()`/`lane_lit_until`/`combo_bump_until`을 정의만 하고 실제 그리기에서 쓰는 코드가 없음 → 통합 루프의 그리기 단계에 적용 예시 포함.

### 적용한 보완 (Round 1)
- 12.2 앞: `Note` 클래스 정의(`.lane/.time/.hit`)와 11장 노트 데이터로 `notes` 리스트를 준비하는 코드 추가.
- 12.2 입력 스니펫: `judge(lane, press_time, notes)` 3인자로 통일, `score, combo = handle_result(...)` 반환 받도록 수정, lane 강조 타이머 설정을 호출부로 명시.
- 12.3: 표/본문에 키입력 MISS(±100ms 초과)와 자동 MISS(통과 후 200ms)를 분리 설명. 그림 12.7 캡션도 두 경계를 구분하도록 정리. 상수 주석에 `MISS_MS`는 자동 MISS 전용임을 명시.
- 12.5 `handle_result()`: `(result, score, combo)` → `(score, combo)` 반환으로 정의 정리(기존 호출 규약과 일치).
- 12.6 자동 MISS: `elapsed` 재대입·중복 `not note.hit` 제거, `current_time - note.time > MISS_MS`로 정리. 자동 MISS도 `combo`/판정글자 타이머를 갱신하도록 유지.
- 12.6 뒤 신설(12.7 "전체 코드 합치기"): 상태 변수 초기화 + 입력/자동MISS/그리기(레인 강조·콤보 폰트·콤보 범프 적용 포함)를 하나로 합친 완성 게임 루프 추가. 이후 결과 확인/트러블슈팅/마무리 절 번호 +1.
- 유지: 아이콘 코너, 그림 `<!-- FIG -->` 마커·번호·순서 모두 그대로.

## Round 2 (2026-06-18) — 실행 가능성 정밀 검증

**판정: fixed** (통합 루프 단독 실행 가능 + 이중 MISS 회색지대 명문화 + 상수 import 명시)

### blocker
1. **12.7 통합 루프가 `pygame.init()`/`screen`/`clock`/`notes` 초기화 없이 사용** — 기존 12.7 도입 블록은 `score`·`judge_text_until` 등 상태 변수만 초기화하고, 정작 루프 본문이 쓰는 `screen.fill(...)`/`clock.tick(60)`/`pygame.display.flip()`/`for note in notes`의 `screen`·`clock`·`notes`는 어디서도 만들지 않음. 12.7만 복붙하면 즉시 `NameError`. → 통합 블록 안에 (1) `import pygame` + `from constants import *` + `pygame.init()`/`set_mode`/`Clock`, (2) `note_data` → `notes = [Note(...)]` 준비, (3) 상태 변수 초기화를 한곳에 모아 "이대로 복사해도 실행됨"을 명시. 루프 끝에 `pygame.quit()` 추가. 실제 헤드리스(SDL dummy)로 추출 실행해 NameError 없이 도는 것 확인.

### major (이중 MISS 회색지대)
2. **키입력 MISS(±100ms 초과) + 자동 MISS(통과 후 200ms)가 한 노트의 콤보를 이중으로 끊는 회색지대** — `judge()`는 키입력 MISS일 때 `closest.hit`을 건드리지 않고 `"MISS", None`을 반환(먼 미래 노트를 삼키지 않기 위한 의도된 동작). 그 결과 같은 노트가 200ms 뒤 자동 MISS로 다시 콤보를 끊고 MISS 글자를 또 띄움. → 동작은 그대로 두되(키 MISS가 노트를 소비하지 않는 게 옳음), 12.6에 ⚠️ 주의 박스로 두 MISS의 관계를 명문화: "노트를 최종적으로 닫고 콤보를 끊는 책임은 항상 자동 MISS 쪽, 키입력 MISS는 즉각 피드백 글자만". 추출 실행로 키 MISS 후 `note.hit==False` 유지 확인.

### minor
3. **`from constants import *` 누락** — 12.3에서 상수를 `constants.py`로 빼라고 권하면서 import 한 줄이 없었음. → 12.3 상수 블록 뒤 한 줄 + 12.7 통합 블록 맨 위 `from constants import *` 명시(같은 파일에 두면 불필요함도 병기).

### 적용한 보완 (Round 2)
- 12.3: 상수 블록 뒤 `from constants import *` 사용법 한 줄 추가(별도 파일/동일 파일 경우 구분).
- 12.6: ⚠️ 주의 박스 신설 — 키입력 MISS와 자동 MISS의 관계(자동 MISS가 노트 종료·콤보 단절의 단일 책임) 한 단락 설명.
- 12.7: 통합 루프 도입 블록을 self-contained하게 재작성 — `import pygame`/`from constants import *`/`pygame.init()`/`set_mode((480,800))`/`Clock()` + `note_data`→`notes` 준비 + 상태 변수 초기화를 한 블록에 포함, 루프 끝에 `pygame.quit()` 추가. "이대로 복사해도 실행됨" 명시.
- 검증: 코드 블록 전체를 추출해 SDL dummy 드라이버로 실제 실행 → `INTEGRATION LOOP RAN OK`, NameError 없음. 키입력 MISS 후 `note.hit==False` 유지 확인.
- 유지: 절 번호(12.1~12.9)·그림 `<!-- FIG -->` 마커·번호(12.1~12.18, 18개)·순서, 아이콘 코너 모두 그대로. PROGRESS.md 미수정.

## Round 3 (2026-06-18) — 그리기 함수 미정의 + 키MISS 모순 수정

**판정: fixed** (5개 그리기 함수 실제 정의로 NameError 제거 + 키입력 MISS 콤보 처리 본문-코드 일치)

### blocker
1. **12.7 통합 루프가 `draw_notes`/`draw_lane_line`/`draw_score`/`draw_combo`/`draw_judge_text` 5개를 정의 없이 호출** — 본문은 "9~11장에서 만든 함수"라고 가정만 했고 이 장 어디에도 정의가 없음. grader가 PASS였던 이유는 통합 블록의 `from constants import *`(star-import)가 `[CODE] 2차` 미정의-이름 검사를 통째로 건너뛰게 만들기 때문(`if not star:`). 즉 grader가 실제 NameError를 가려 줬을 뿐, 복붙 시 런타임 NameError. → 12.7 도입부에 그리기 함수 5개를 **실제로 정의**(pygame.draw.rect + SysFont.render). 12.7 루프 호출 시그니처와 인자 순서를 1:1로 맞춤(`draw_lane_line(screen, lane, bright=lit)` 등). 통합 준비 블록도 self-contained하게 `Note`/`KEY_LANE`/`SCORE_TABLE`/`judge()`/`handle_result()`/`combo_font_size()`를 한곳에 모아 단독 실행 보장.

### major (본문-코드 모순)
2. **키입력 MISS 콤보 처리 모순** — 12.6 ⚠️ 주의 박스 본문은 "키입력 MISS는 즉각 피드백 글자만 띄울 뿐, 콤보를 끊는 책임은 자동 MISS에 있다"고 했으나, 코드상 `handle_result("MISS", ...)`는 키입력 MISS에서도 `combo = 0`으로 즉시 끊음. 본문↔코드 충돌. → 코드를 진실로 삼아 **본문을 코드에 일치**: "콤보는 키입력 MISS·자동 MISS 두 경우 모두 끊긴다. 차이는 노트 소비(`hit=True`) 여부뿐 — 노트를 최종적으로 닫는(소비하는) 책임만 자동 MISS 쪽"으로 수정. (`handle_result` 코드는 변경하지 않음.)

### 적용한 보완 (Round 3)
- 12.7: 그리기 함수 5개(`draw_notes`/`draw_lane_line`/`draw_score`/`draw_combo`/`draw_judge_text`) + 좌표 헬퍼(`lane_x`)·상수(`LANE_W`/`JUDGE_Y`/`LANE_COLOR`/`JUDGE_COLOR`) 실제 정의 블록 신설. 노트=사각형, 판정선=가로 막대, 점수/콤보/판정글자=`SysFont.render`. 콤보 100+ 색 강조, 판정 글자 색(PERFECT 노랑/GOOD 파랑/MISS 빨강) 반영.
- 12.7: 통합 준비 블록을 단독 실행 가능하도록 재작성 — `Note`/`KEY_LANE`/`SCORE_TABLE`/`judge()`/`handle_result()`/`combo_font_size()`를 (0)번 블록으로 모음. 도입/마무리 문장도 "방금 정의한 5개"로 정정(기존 "9~11장에서 만든" 가정 문구 수정).
- 12.6: ⚠️ 주의 박스 본문을 코드와 일치 — "콤보는 두 MISS 모두 끊긴다, 차이는 노트 소비 여부"로 정리.
- 검증: 그리기 블록 + 준비 블록 + while 블록을 추출해 SDL/audio dummy로 헤드리스 실행 → `INTEGRATION LOOP RAN OK`, NameError 0. 별도 단위 확인: 키입력 MISS 후 `note.hit==False` 유지 + `handle_result` 콤보 0으로 끊김, PERFECT 시 `note.hit==True`.
