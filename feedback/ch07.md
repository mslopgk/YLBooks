# ch07 피드백 — 채터링과의 전쟁(디바운싱)

## Round 1 (2026-06-18) — fresh 학습자 검증

**판정: needs-work** (스케치 폴더/파일 규칙·라이브러리 설치·출력 검증 절차 누락으로 fresh 학습자가 컴파일·검증 단계에서 막힘)

### major
1. **스케치 폴더명 = `.ino` 파일명 규칙 미명시** — 7.2 `Chatter` 폴더를 만들라고 하면서 폴더 이름과 `.ino` 파일 이름이 같아야 한다는 arduino-cli 규칙을 안 알려줌. 규칙을 모르면 `Chatter.ino`가 아닌 다른 이름으로 저장 → 컴파일 실패. (5장 Blink에서는 명시했으나 7장은 자기완결성 부족) → 7.2에 규칙 한 줄 추가.
2. **`Keyboard` 라이브러리 설치 명령 부재** — 7.6 결과 코드가 `#include <Keyboard.h>`를 쓰는데 이 챕터엔 설치 안내가 없음. 5장을 건너뛰었거나 환경을 새로 잡은 학습자는 컴파일 에러. → "5장에서 설치" 명시 + 미설치 시 `arduino-cli lib install "Keyboard"` 한 줄 안내.
3. **리팩토링 결과물 저장 위치/시점 불명확** — 7.7이 갑자기 "폴더 이름이 `Bounce`라면"이라고 하는데, `Bounce` 폴더를 언제 어떻게 만드는지 본문에 없음. 7.6 리팩토링 결과를 어디에 저장하는지 연결이 끊김. → 7.6 끝에 `Bounce/Bounce.ino`로 저장하라는 안내 추가, 7.7 도입부에서 그 폴더를 가리키도록 정리.
4. **연습문제1이 115200 baud 고속 출력이라 육안 카운트 불가** — "버튼 한 번에 `0`이 몇 번 찍히는지 세어 보라"는데 빠르게 흘러내려 사람이 못 셈. → 출력을 파일로 저장(`>` 리다이렉트)해서 세는 방법을 안내하도록 문제 수정.

### minor
- 7.3 `millis()` 예제가 `setup()` 없는 불완전 스니펫인데 "그대로 빌드하지 말라"는 안내가 없음 → 학습용 발췌임을 한 줄 명시.
- 7.7 팁("시리얼 출력은 업로드 전에 정리")이 최종 Bounce2 코드와 어긋남 — 최종 코드엔 이미 `Serial`이 없음. 팁이 "채터링 관찰 코드 → 디바운싱 코드로 넘어갈 때 Serial을 안 들고 온다"는 의미가 되도록 문구 정리.
- 풀업(`INPUT_PULLUP`)에서 눌리면 LOW가 된다는 전제가 본론에 없이 7.2 코드부터 LOW를 당연시함 → 7.2에 풀업=평상시 HIGH, 눌림 LOW 한 줄 보충.

### 적용한 보완 (Round 1)
- 7.2: 스케치 폴더명 = `.ino` 파일명 규칙 한 줄 추가(5장과 동일 표현). 풀업 전제(평상시 HIGH, 눌림 LOW) 한 줄 보충.
- 7.3: `millis()` 스니펫이 학습용 발췌이며 그대로 빌드하지 말라는 안내 한 줄 추가.
- 7.4 인근: `Keyboard` 라이브러리는 5장에서 설치했음을 명시하고, 안 했으면 `arduino-cli lib install "Keyboard"`로 받으라는 안내 추가.
- 7.6: 리팩토링 결과를 `Bounce` 폴더 안 `Bounce.ino`로 저장하라는 안내 추가(폴더명=파일명 규칙 재확인).
- 7.7: 도입부를 `Bounce` 폴더를 컴파일·업로드하는 흐름으로 정리. Serial 정리 팁을 "관찰 코드와 디바운싱 코드를 분리한다"는 맞는 의미로 수정.
- 연습문제1: `arduino-cli monitor ... > chatter.txt`로 출력을 파일에 저장한 뒤 `0`의 개수를 세는 방식으로 문제 수정.

## Round 2 (2026-06-18) — fresh 학습자 재검증

**판정: needs-work** (Round 1에서 추가한 `> chatter.txt` 리다이렉트가 실제 환경에서 동작 보장 안 됨, 폴더명 혼동 잔존)

### major
1. **연습문제1의 `arduino-cli monitor ... > chatter.txt` 리다이렉트가 검증 안 됨** — `monitor`는 대화형 스트림이라 PowerShell/환경에 따라 파일이 비거나, `Ctrl+C` 후에도 깔끔히 안 멈추거나, 버퍼링으로 0 카운트가 누락될 수 있음. fresh 학습자가 빈 파일을 열고 막힘. → 검증 안 된 리다이렉트 제거. 모니터를 띄운 채 천천히 한 번 누르고 화면을 캡처/눈으로 관찰해 `0` 개수를 가늠하는, 실제로 되는 방법으로 교체.
2. **스케치 폴더명 `Bounce`가 라이브러리 `Bounce2`/클래스 `Bounce`와 혼동** — 7.6/7.7에서 만드는 폴더 `Bounce`가 코드 속 `Bounce` 클래스, 설치한 `Bounce2` 라이브러리와 이름이 겹쳐 "내 코드인지 라이브러리인지" 헷갈림. → 폴더·파일명을 `BounceButton`으로 변경(7.6 저장 안내, 7.7 compile/upload 명령, 7.7 팁 본문까지 일괄). 왜 `Bounce`를 피하는지 한 줄 근거 추가.

### minor
- 포트(COM) 식별법이 "`board list`로 확인" 한마디뿐 → 7.2에 "보드 이름이 적힌 줄의 맨 앞 `COMx`가 내 포트"라는 고르는 법 한 줄 추가.
- 연습문제3 FQBN 알아내는 법 부재 → `arduino-cli board listall`로 자기 보드 줄의 FQBN을 쓰라는 안내 추가.
- INPUT_PULLUP=평상시 HIGH/눌림 LOW 전제는 7.2 본론(7.2 도입 코드 설명 단락)에 이미 명시되어 있어 추가 불필요(확인 완료).

### 적용한 보완 (Round 2)
- 연습문제1: `> chatter.txt` 리다이렉트 삭제 → 모니터 띄운 채 천천히 1회 누름 + 화면 캡처/육안 관찰로 `0` 개수 가늠하는 방식으로 교체(검증된 동작).
- 7.6/7.7: 스케치 폴더·파일명 `Bounce`→`BounceButton` 일괄 변경(저장 안내, compile/upload 두 명령, Serial 정리 팁 본문). `Bounce`를 피하는 이유(라이브러리/클래스명 충돌) 한 줄 추가.
- 7.2: `board list` 출력에서 내 포트(보드 이름 줄의 `COMx`) 고르는 법 한 줄 추가.
- 연습문제3: `board list`(포트)·`board listall`(FQBN)로 값 찾는 법 안내 추가.
- 유지: 아이콘 코너, 그림 `<!-- FIG -->` 마커·번호·순서, 표 번호, 친근한 대화체 모두 그대로. PROGRESS.md 미수정.

## Round 3 (2026-06-18) — 잔존 결함 정밀 수정

**판정: needs-work** (Round 2의 "육안 관찰" 방식이 115200 고속 출력에서는 여전히 카운트 불가, 연습문제3의 `listall` 안내가 부정확)

### major
1. **연습문제1 — 115200 고속 출력은 사람이 못 셈** — Round 2에서 "모니터 띄운 채 천천히 1회 누르고 화면 캡처/육안 관찰로 `0` 개수 가늠"으로 고쳤지만, `delay` 없는 관찰용 코드는 초당 수천 줄을 쏟아내 화면을 캡처해도 어느 줄이 한 번의 누름인지 분간이 안 됨. → 관찰용 코드의 `Serial.println(val);` 아래에 `delay(2);`를 넣어 2ms마다 한 줄씩만 출력하도록 살짝 바꿔, 사람이 `0` 줄 수를 실제로 셀 수 있게 교체. `0` 한 줄≈2ms라는 환산까지 곁들여 `interval()` 값 가늠과 연결. 다 센 뒤 `delay(2);`를 지워 원복하라는 안내 추가.

2. **연습문제3 — `board listall`로 "내 FQBN 찾기"는 부정확** — `board listall`은 연결된 내 보드를 짚어 주는 게 아니라 설치된 모든 보드 정의를 나열하는 명령. fresh 학습자가 긴 목록에서 헤맴. → FQBN은 우리 보드가 Arduino Micro이므로 `arduino:avr:micro`를 그대로 쓰면 된다고 못 박고, `listall`은 "내 보드를 찾아 주는 명령이 아니다"라고 경고. 포트(COMx)는 장치마다 달라 `board list`로 확인한다는 구분을 명확히(보드 이름 줄의 맨 앞 `COMx`가 내 포트).

### 적용한 보완 (Round 3)
- 연습문제1: 관찰용 코드에 `delay(2);` 한 줄을 추가해 출력을 2ms 간격으로 늦춰 `0` 줄 수를 육안으로 세는 방식으로 교체. `0` 한 줄≈2ms 환산으로 떨림 길이·`interval()` 값 가늠 연결. 측정 후 `delay(2);` 원복 안내 포함.
- 연습문제3: FQBN=`arduino:avr:micro`(우리 보드=Micro) 고정 안내 + `board listall`이 "전체 정의 나열"이라 "내 보드 찾기"가 아님을 경고. 포트는 `board list`로 찾고, 보드 이름 줄 맨 앞 `COMx`가 내 포트라는 구분 명확화.
- 유지: 아이콘 코너, 그림 `<!-- FIG -->` 마커·번호·순서, 표 번호, 친근한 대화체 모두 그대로. PROGRESS.md 미수정. (앞 장에서 익힌 arduino-cli·Claude Code 전제는 결함 아니므로 손대지 않음.)
